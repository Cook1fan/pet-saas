# 宠物门店私域运营一体化 SaaS

> 一站式宠物门店数字化解决方案，包含门店管理、会员体系、商品库存、订单收银、营销活动等核心功能。

---

## 功能概述

### 已实现功能

#### 后端 API (28 个 Controller，31 个 Service)

| 模块 | 控制器 | 功能 |
|-----|-------|------|
| **商品管理** | GoodsController | 商品列表、详情、录入、编辑、上下架、库存预警、变更历史 |
| **分类管理** | GoodsCategoryController | 商品分类树 |
| **库存管理** | StockController | 手动入库/出库、盘点调整、库存流水 |
| **订单管理** | OrderController | 开单收银、订单列表、订单详情、退款 |
| **会员管理** | MemberController | 会员列表、录入编辑、储值账户、次卡管理 |
| **宠物管理** | PetController | 宠物档案管理 |
| **拼团活动** | ActivityController, GroupActivityService | 拼团/秒杀活动创建、管理、数据统计 |
| **储值&次卡** | RechargeRuleMapper, CardRuleMapper | 储值规则、次卡规则配置 |
| **二维码** | QrCodeController, ShopQrCodeController | 门店二维码生成、统计、下载 |
| **微信支付** | WechatPayController | 支付、退款 |
| **对账中心** | FlowRecordService | 流水记录、自动对账、异常告警 |
| **文件上传** | FileUploadController | 文件上传服务 |
| **AI 功能** | UserAiController | AI 文案生成、问答配置 |
| **租户管理** | TenantController | 多租户管理 |
| **认证** | AuthController | PC端登录认证 |

#### 前端页面 (40+ 个页面)

| 模块 | 页面 |
|-----|------|
| **平台端** | 登录、仪表盘、租户列表、AI配置、短信配置 |
| **门店PC端** | 登录、仪表盘、收银台、订单列表/详情、会员列表/详情/导入、商品管理、库存记录、储值规则、次卡规则、储值记录、拼团活动、秒杀、限时抢购、营销数据、门店信息、员工管理、支付配置、AI文案、AI配额、流水对账、对账详情、二维码 |

#### 小程序页面 (10+ 个页面)

| 模块 | 页面 |
|-----|------|
| **用户端** | 首页、用户登录、选择门店、用户中心 |
| **拼团** | 拼团列表、拼团详情、团详情、拼团确认、拼团成功、我的订单 |
| **二维码** | 二维码入口 |

## 快速启动

### 环境要求

| 工具 | 版本 |
|-----|------|
| JDK | 21+ |
| Maven | 3.8+ |
| Node.js | 18+ |
| MySQL | 8.0+ |

### 环境变量配置

敏感配置（密码、API Key 等）通过环境变量注入，不会提交到 Git。

```bash
# 后端：复制模板文件并填入真实值
cd pet-saas-backend
cp .env.example .env
# 编辑 .env 填入真实配置

# 启动时加载环境变量
export $(cat .env | xargs) && mvn spring-boot:run
```

详细配置请参考 [pet-saas-backend/.env.example](pet-saas-backend/.env.example)

### 后端启动

```bash
cd pet-saas-backend
# 首次运行需要初始化数据库
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

### 前端启动

```bash
cd pet-saas-admin-pc
npm install
npm run dev
```

### 小程序启动

```bash
cd pet-saas-miniapp
npm install
# 使用 HBuilderX 打开项目运行
```

## 开发规范

- **API 优先**：所有接口先定义契约再编码
- **文档同步**：代码变更需同步更新文档
- **多租户隔离**：所有数据查询必须带 tenant_id 条件
- **编码规范**：私有方法调用必须使用 `this.` 前缀
