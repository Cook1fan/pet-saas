# 开发流程

## Phase 概览

### Phase 1: 理解需求

1. 阅读 `docs/` 下的相关 SPEC 文档
2. 确认功能需求与验收标准
3. 查看后端接口文档

### Phase 2: 设计实现

1. 设计页面结构与组件拆分
2. 定义类型定义 (types.ts)
3. 设计 Store 状态结构
4. 设计接口调用

### Phase 3: 编码实现

1. 创建页面文件
2. 实现 API 封装
3. 实现状态管理
4. 实现页面逻辑

### Phase 4: 自测验证

1. 功能自测
2. 对照验收标准检查
3. 与后端联调

## 页面开发 Checklist

- [ ] 路由已添加到 router
- [ ] API 接口已封装
- [ ] 类型定义已写好
- [ ] 表单验证已实现
- [ ] 错误处理已完善
- [ ] 加载状态已添加
- [ ] 权限控制已处理
