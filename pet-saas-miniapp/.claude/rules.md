# 小程序强制编码规则

## 包体积优化

- ✅ 图片资源必须压缩后上传 CDN，禁止直接放入项目
- ✅ 按需引入组件，避免全量引入 UI 库
- ✅ 合理使用分包，非首屏页面放入分包
- ❌ 禁止引入超过 100KB 的单文件依赖

## 性能要求

- ✅ 列表必须使用虚拟滚动（uni-list 或自定义）
- ✅ 图片使用 `lazy-load` 懒加载
- ✅ 避免频繁 setData，批量更新数据
- ✅ 使用 `onPageScroll` 节流

## 兼容性

- ✅ 使用 uni API 而非 wx 原生 API
- ✅ 检查基础库版本，低版本做降级处理
- ✅ 测试 iPhone SE（小屏）和 iPhone Pro Max（大屏）适配

## 安全

- ✅ 用户敏感数据必须加密存储
- ✅ 登录态使用 uni.setStorageSync 加密存储
- ✅ 接口请求必须携带 token 且验证签名
