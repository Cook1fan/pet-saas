# API 契约中心

> 🔴 **重要**：所有接口先在此定义，前后端确认后再实现

---

## 📋 目录

- [接口规范.md](./接口规范.md) - 通用接口规范
- [types.ts](./types.ts) - TypeScript 类型定义（前后端共用）

---

## 🎯 如何使用

### 新功能开发流程

1. **在此定义接口**
   - 编写 `types.ts` 定义请求/响应类型
   - 在对应模块文档中描述接口细节

2. **前后端确认**
   - 在根目录 Claude 中讨论确认
   - 有分歧及时调整

3. **并行开发**
   - 后端：基于契约实现
   - 前端：基于契约 + Mock 开发

---

## 📝 类型定义说明

`types.ts` 是前后端共用的类型定义：
- 后端可以用工具生成 Java DTO
- 前端直接用于 TypeScript 类型检查
- 保持单一真实来源
