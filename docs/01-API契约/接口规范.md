# 通用接口规范

> 本文档定义所有 API 的通用规范

---

## 🔌 基础信息

- **Base URL**: `/api`
- **请求格式**: JSON
- **响应格式**: JSON
- **字符编码**: UTF-8

---

## 📨 请求头

所有需要认证的接口需携带：

| Header | 说明 | 示例 |
|--------|------|------|
| `satoken` | Sa-Token | `pc_xxxxx` |
| `tenant-id` | 租户 ID（门店端） | `1` |

---

## 📤 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1712345678901
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | number | 状态码，200=成功 |
| `message` | string | 提示信息 |
| `data` | any | 响应数据 |
| `timestamp` | number | 时间戳 |

---

## 🎯 状态码

| Code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

---

## 📄 分页请求

```json
{
  "page": 1,
  "pageSize": 10,
  "condition": {}
}
```

## 📄 分页响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```
