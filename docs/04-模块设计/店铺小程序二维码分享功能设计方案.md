## 1. 业务概述

### 1.1 功能背景

店铺需要一个专属的推广二维码，可以让客户扫码直接进入该店铺的小程序。新客户扫码后完成注册/登录，老客户直接跳转到店铺首页。

### 1.2 核心功能点

| 功能点 | 说明 |
|--------|------|
| 二维码生成 | 店铺生成专属的小程序码，带有租户标识 |
| 扫码识别 | 客户扫码后，小程序识别租户信息 |
| 新客户引导 | 未登录客户引导完成注册/登录 |
| 老客户跳转 | 已登录/已注册客户直接进入店铺首页 |
| 二维码管理 | 管理后台可查看/下载/刷新二维码 |

### 1.3 技术方案选择

微信小程序支持两种二维码生成方式：

| 方式 | 接口 | 限制 | 适用场景 |
|------|------|------|----------|
| 永久二维码 | `createQRCode` | 有限数量（最多10万个） | 不适用 |
| 永久二维码（无数量限制） | `createLimitQRCode` | 需要 scene 参数（最大32字符） | 场景受限 |
| 云托管静态网站二维码 | 插件提供 | 无数量限制 | 推荐 |

**推荐方案：云开发静态网站二维码**
- 使用微信云开发能力生成无数量限制的二维码
- scene 参数存储租户标识（格式：`tenant_{tenantId}`）
- 可通过云开发控制台管理

### 2.2 新客户首次扫码流程

```
1. 客户扫描店铺二维码
   ↓
2. 小程序启动，解析 scene 参数获取 tenantId
   ↓
3. 调用后端接口验证租户有效性
   ↓
4. 检查本地缓存是否有有效登录态
   ├─ 无登录态 → 进入注册引导页
   ↓
5. 引导页显示：
   - 店铺名称和 Logo
   - "欢迎来到 {店铺名称}" 文案
   - 微信手机号一键注册（优先）
   - 或输入手机号+验证码注册
   ↓
6. 用户完成注册/登录
   ↓
7. 绑定会员与 tenant 关系（自动创建会员档案）
   ↓
8. 进入店铺首页
```

### 2.3 老客户扫码流程

```
1. 客户扫描店铺二维码
   ↓
2. 小程序启动，解析 scene 参数获取 tenantId
   ↓
3. 调用后端接口验证租户有效性
   ↓
4. 检查本地缓存是否有有效登录态
   ├─ 有登录态 → 查询会员信息
   │    ↓
   │    验证会员是否属于该租户
   │    ├─ 是 → 直接进入店铺首页
   │    └─ 否 → 提示"该码仅限 {店铺名} 客户使用"
   │
   └─ 无登录态 → 引导登录
        ↓
   登录成功后验证租户归属
```

### 2.4 游客访问流程（可选）

对于不想注册的客户，也希望能浏览店铺：

```
1. 扫码 → 验证租户 → 无登录态
   ↓
2. 选择"游客体验"
   ↓
3. 进入店铺首页（浏览态）
   ↓
4. 下单/预约时提示注册
```

## 3. 数据库设计

### 3.1 新增表：shop_qr_code（店铺二维码表）

```sql
CREATE TABLE `shop_qr_code` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键 ID',
  `tenant_id` bigint NOT NULL COMMENT '租户 ID',
  `qr_type` tinyint NOT NULL DEFAULT 1 COMMENT '二维码类型：1-店铺码，2-商品码，3-活动码',
  `qr_name` varchar(100) NOT NULL COMMENT '二维码名称/描述',
  `scene` varchar(64) NOT NULL COMMENT '场景参数（如 tenant_123）',
  `qr_url` varchar(500) COMMENT '二维码图片 URL',
  `qr_ticket` varchar(256) COMMENT '微信返回的 ticket',
  `expire_time` datetime COMMENT '过期时间（永久二维码则为空）',
  `is_default` tinyint NOT NULL DEFAULT 0 COMMENT '是否默认二维码：0-否，1-是',
  `scan_count` int NOT NULL DEFAULT 0 COMMENT '扫码次数',
  `create_user` bigint DEFAULT NULL COMMENT '创建人 ID',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
  `update_user` bigint DEFAULT NULL COMMENT '修改人 ID',
  `is_deleted` tinyint NOT NULL DEFAULT 0 COMMENT '逻辑删除：0-未删除，1-已删除',
  PRIMARY KEY (`id`),
  KEY `idx_tenant_id` (`tenant_id`),
  KEY `idx_qr_type` (`qr_type`),
  KEY `idx_scene` (`scene`),
  KEY `idx_is_default` (`is_deleted`, `tenant_id`, `is_default`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='店铺二维码表';
```


```
  "code": 200,

    "qrCodeId": 1,
```

1. 查询该租户是否已有默认二维码

```javascript
// pages/qrcode-entry/qrcode-entry.js
Page({
  data: {
    tenantInfo: null,
    isLoading: true
  },

  onLoad(options) {
    const scene = decodeURIComponent(options.scene || '')
    if (scene && scene.startsWith('tenant_')) {
      const tenantId = scene.split('_')[1]
      this.verifyQrCode(tenantId)
    }
  },

  async verifyQrCode(tenantId) {
    try {
      const res = await wx.request({
        url: '/api/user/qrcode/verify',
        data: { scene: `tenant_${tenantId}` }
      })

      if (res.data.code === 200) {
        this.setData({
          tenantInfo: res.data.data,
          isLoading: false
        })

        const token = wx.getStorageSync('token')
        if (token) {
          this.checkUserBinding(tenantId)
        }
      }
    } catch (e) {
      wx.showToast({ title: '二维码无效', icon: 'none' })
    }
  },

  async checkUserBinding(tenantId) {
    // 检查当前用户是否已绑定该店铺
    // 如果已绑定，直接跳转首页
    // 如果未绑定，显示引导注册页
  }
})
```

## 7. 技术实现方案

### 7.1 微信小程序码生成

```java
// QrCodeServiceImpl.java
@Service
@RequiredArgsConstructor
public class QrCodeServiceImpl implements QrCodeService {

    private final WxMaService wxMaService;
    private final ShopQrCodeMapper shopQrCodeMapper;

    @Override
    public String generateShopQrCode(Long tenantId, String qrName) {
        // 1. 检查是否已存在默认二维码
        ShopQrCode existingQr = shopQrCodeMapper.selectOne(
            new LambdaQueryWrapper<ShopQrCode>()
                .eq(ShopQrCode::getTenantId, tenantId)
                .eq(ShopQrCode::getQrType, 1)
                .eq(ShopQrCode::getIsDefault, 1)
                .eq(ShopQrCode::getIsDeleted, 0)
        );

        if (existingQr != null) {
            return existingQr.getQrUrl();
        }

        // 2. 调用微信接口生成永久二维码
        String scene = "tenant_" + tenantId;
        String ticket = wxMaService.getQrCodeService().createLimitQRCode(scene);

        // 3. 获取二维码图片 URL
        String qrUrl = "https://api.weixin.qq.com/cgi-bin/showqrcode?ticket=" + ticket;

        // 4. 保存到数据库
        ShopQrCode qrCode = new ShopQrCode();
        qrCode.setTenantId(tenantId);
        qrCode.setQrType(1);
        qrCode.setQrName(qrName);
        qrCode.setScene(scene);
        qrCode.setQrUrl(qrUrl);
        qrCode.setQrTicket(ticket);
        qrCode.setIsDefault(1);
        shopQrCodeMapper.insert(qrCode);

        return qrUrl;
    }
}
```

### 7.2 二维码验证与绑定

```java
@Override
public QrCodeVerifyResp verifyQrCode(String scene) {
    if (!scene.startsWith("tenant_")) {
        throw new BusinessException("无效的二维码");
    }

    Long tenantId = Long.parseLong(scene.split("_")[1]);

    SysTenant tenant = sysTenantMapper.selectById(tenantId);
    if (tenant == null || tenant.getStatus() != 1) {
        throw new BusinessException("店铺不存在或已停用");
    }

    QrCodeVerifyResp resp = new QrCodeVerifyResp();
    resp.setValid(true);
    resp.setTenantId(tenantId);
    resp.setTenantName(tenant.getShopName());
    resp.setTenantLogo(tenant.getLogo());

    return resp;
}

@Override
@Transactional
public void bindTenant(Long memberId, Long tenantId) {
    MemberShopBind existing = memberShopBindMapper.selectOne(
        new LambdaQueryWrapper<MemberShopBind>()
            .eq(MemberShopBind::getMemberId, memberId)
            .eq(MemberShopBind::getTenantId, tenantId)
    );

    if (existing != null) {
        // 更新最后访问时间
        existing.setLastVisitTime(new Date());
        existing.setBindSource(1); // 扫码注册
        memberShopBindMapper.updateById(existing);
        return;
    }

    MemberShopBind bind = new MemberShopBind();
    bind.setMemberId(memberId);
    bind.setTenantId(tenantId);
    bind.setBindSource(1); // 扫码注册
    memberShopBindMapper.insert(bind);

    // 更新会员来源信息
    Member member = memberMapper.selectById(memberId);
    member.setRegisterSource(1);
    member.setFromTenantId(tenantId);
    memberMapper.updateById(member);

    logScan(tenantId, memberId, 1);
}
```

## 8. 风险评估

### 8.1 技术风险

| 风险 | 等级 | 说明 | 应对措施 |
|------|------|------|----------|
| 微信二维码接口调用频率限制 | 中 | 微信对生成二维码有频率限制 | 缓存已生成的二维码，避免重复生成 |
| 二维码过期 | 低 | 微信返回的 ticket 有有效期 | 定期刷新并更新本地存储 |
| 会员绑定关系丢失 | 中 | 删除绑定关系后会员无法访问店铺 | 提供"解绑后重新绑定"入口 |
| 恶意刷码 | 高 | 竞争对手可能恶意刷码 | 添加 IP 限流，同一 IP 每分钟最多 10 次 |

### 8.2 业务风险

| 风险 | 等级 | 说明 | 应对措施 |
|------|------|------|----------|
| 用户不愿意注册 | 高 | 强制注册可能导致用户流失 | 提供游客浏览模式，注册仅在下单时要求 |
| 店铺注销后二维码仍可访问 | 低 | 租户注销后二维码 scene 仍有效 | 小程序端验证租户状态，不存在则提示错误 |
| 跨店访问 | 中 | 会员可能在多个店铺注册 | 通过 `member_shop_bind` 区分访问权限 |
| 二维码被滥用 | 中 | 二维码被贴到其他店铺/场所 | 二维码与店铺 ID 绑定，扫码记录作为溯源依据 |

### 8.3 安全考虑

1. **scene 参数安全**：scene 只存储租户 ID，不包含敏感信息
2. **防重放攻击**：二维码验证接口添加时间戳校验
3. **防爬虫**：验证接口添加频率限制
4. **日志审计**：记录所有扫码行为，便于追溯
