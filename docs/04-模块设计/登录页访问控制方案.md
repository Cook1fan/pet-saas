# 登录页访问控制方案

## 1. 问题背景

### 1.1 现状
- 商家端登录页：`/shop/login`
- 平台端登录页：`/platform/login`
- 两个登录页目前在开发和生产环境都可以自由访问

### 1.2 需求
| 环境 | 用户角色 | 商家端登录 | 平台端登录 |
|------|----------|-----------|-----------|
| 开发环境 | 开发者 | ✅ 可访问 | ✅ 可访问 |
| 生产环境 | 商家用户 | ✅ 可访问 | ❌ 不可访问 |
| 生产环境 | 平台管理员 | ✅ 可访问 | ✅ 可访问 |

---

## 2. 可选方案对比

### 方案一：环境变量控制（最简单）

**原理**：通过 `import.meta.env.DEV` 判断环境，生产环境隐藏平台端入口链接

| 维度 | 评价 |
|------|------|
| 实现难度 | ⭐ 5分钟 |
| 安全性 | ⭐⭐ 知道 URL 仍可访问 |
| 维护成本 | ⭐⭐⭐⭐⭐ 几乎无需维护 |
| 推荐度 | ⭐⭐ 临时方案 |

**优点**：
- 实现最简单，只需改几行代码

**缺点**：
- 安全性不足，商家若知道 `/platform/login` 仍可直接访问
- 只是"隐藏"而非"禁止"

---

### 方案二：路由守卫 + 特殊访问参数（推荐）

**原理**：
- 商家端登录页不显示平台端入口链接
- 平台端路由加守卫：
  - 开发环境：直接放行
  - 生产环境：检查 URL 是否包含 `?admin=1` 参数，否则跳转商家端
- 平台管理员记住特殊参数：`/platform/login?admin=1`

| 维度 | 评价 |
|------|------|
| 实现难度 | ⭐⭐ 30分钟 |
| 安全性 | ⭐⭐⭐ 一般商家不知道参数 |
| 维护成本 | ⭐⭐⭐⭐ 只需记住参数 |
| 推荐度 | ⭐⭐⭐⭐ 现阶段最佳 |

**优点**：
- 实现简单，安全性较好
- 不需要额外后端支持
- 管理员只需记住一个简单参数

**缺点**：
- 特殊参数可能泄露（但影响可控）
- 不是"真正"的安全验证

---

### 方案三：子域名分离（最规范）

**原理**：
- `admin.example.com` → 平台管理端
- `shop.example.com` → 商家端
- Nginx/网关层做隔离，互相不可访问

| 维度 | 评价 |
|------|------|
| 实现难度 | ⭐⭐⭐⭐ 需要域名配置 |
| 安全性 | ⭐⭐⭐⭐⭐ 完全隔离 |
| 维护成本 | ⭐⭐⭐ 需要运维配合 |
| 推荐度 | ⭐⭐⭐ 长期方案 |

**优点**：
- 最专业，完全隔离
- 符合 SaaS 最佳实践

**缺点**：
- 需要配置子域名
- 本地开发需要配置 hosts
- 稍显复杂

---

### 方案四：IP 白名单 + 后端验证（最安全）

**原理**：
- 生产环境访问平台端登录页：
  - 前端：先检查特殊参数
  - 后端：验证访问 IP 是否在白名单
- 或者需要先访问一个隐藏的「管理员入口」

| 维度 | 评价 |
|------|------|
| 实现难度 | ⭐⭐⭐⭐ 需要后端配合 |
| 安全性 | ⭐⭐⭐⭐⭐ 最高 |
| 维护成本 | ⭐⭐ IP 可能变化 |
| 推荐度 | ⭐⭐⭐ 安全要求高时用 |

**优点**：
- 安全性最高
- 即使参数泄露，IP 不在白名单也无法访问

**缺点**：
- 管理员 IP 可能变化（家庭宽带、出差等）
- 需要后端配合开发
- 维护成本较高

---

## 3. 推荐方案：方案二（路由守卫 + 特殊参数）

### 3.1 选择理由
- 平衡了安全性和便利性
- 适合当前 MVP 阶段
- 未来可平滑升级到方案三或方案四

### 3.2 实现细节

#### 3.2.1 前端实现

**1. 修改商家端登录页**
- 生产环境隐藏「平台管理端登录」链接
- 开发环境保留

**2. 添加平台端路由守卫**
```typescript
// router/index.ts 中添加
beforeEnter: (to, from, next) => {
  const isDev = import.meta.env.DEV
  const hasAdminParam = new URLSearchParams(window.location.search).has('admin')
  
  if (isDev || hasAdminParam) {
    next()
  } else {
    next('/shop/login')
  }
}
```

**3. 平台端登录页提示**
- 生产环境如果直接访问，提示需要特殊参数
- 或者静默跳转（体验更好）

#### 3.2.2 管理员使用方式
- 开发环境：`/platform/login`
- 生产环境：`/platform/login?admin=1`
