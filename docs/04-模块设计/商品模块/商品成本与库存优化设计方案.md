# 商品成本与库存优化设计方案

> **文档导航：** [返回总览](./00-总览.md) | [核心业务逻辑](./03-核心业务逻辑.md)

---

## 1. 问题背景

### 1.1 需求分析

基于业务讨论，需要对商品模块进行以下优化：

| 需求项 | 说明 |
|--------|------|
| 商品成本价启用 | 成本价从"预留"状态改为正式启用，用于财务核算销售毛利 |
| 库存允许负数 | 实物商品库存允许为负，适应特殊业务场景 |
| 无限库存控制优化 | 用独立属性控制是否无限库存，而不是用 stock=-1 |
| 预留库存设计 | 支持"在途库存"或"预留库存"场景（货未到先销售） |
| 条形码放开使用 | barcode 字段从"预留"改为正式使用 |

---

## 2. 总体设计

### 2.1 核心设计原则

1. **财务友好**：成本价必填，销售时记录成本，支持毛利计算
2. **灵活库存**：支持负库存、无限库存、预留库存多种模式
3. **条码优先**：支持扫码录单、扫码销售
4. **兼容过渡**：保持数据库兼容，平滑迁移

## 3. 数据库设计

### 3.1 goods_sku 表变更

```sql
-- =============================================
-- goods_sku 表结构变更
-- =============================================

-- 1. 修改 cost_price 字段为 NOT NULL（设置默认值 0 用于兼容旧数据）
ALTER TABLE goods_sku
    MODIFY COLUMN cost_price DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '成本价';

-- 2. 新增 is_unlimited_stock 字段（替代 stock=-1 的设计）
ALTER TABLE goods_sku
    ADD COLUMN is_unlimited_stock TINYINT NOT NULL DEFAULT 0 COMMENT '是否无限库存：0-否，1-是' AFTER stock;

-- 3. 新增 reserved_stock 字段（预留/在途库存）
ALTER TABLE goods_sku
    ADD COLUMN reserved_stock INT NOT NULL DEFAULT 0 COMMENT '预留库存（在途库存）' AFTER is_unlimited_stock;

-- 4. 新增 allow_negative_stock 字段（是否允许负库存）
ALTER TABLE goods_sku
    ADD COLUMN allow_negative_stock TINYINT NOT NULL DEFAULT 0 COMMENT '是否允许负库存：0-否，1-是' AFTER reserved_stock;

-- 5. 修改 barcode 字段，增加索引
ALTER TABLE goods_sku
    MODIFY COLUMN barcode VARCHAR(50) COMMENT '商品条码';
ALTER TABLE goods_sku
    ADD INDEX idx_barcode (barcode);

-- 6. 增加 sku_code 唯一索引（租户内唯一）
ALTER TABLE goods_sku
    ADD UNIQUE KEY uk_tenant_sku_code (tenant_id, sku_code);

-- =============================================
-- 数据迁移：将旧数据的 stock=-1 迁移到 is_unlimited_stock=1
-- =============================================
UPDATE goods_sku SET is_unlimited_stock = 1 WHERE stock = -1;
UPDATE goods_sku SET stock = 0 WHERE stock = -1;
```

### 3.2 新增 goods_barcode 表（可选，支持多条码）

**说明**：如果一个商品需要支持多个条码（如不同包装、不同供应商条码），则增加此表。

```sql
-- =============================================
-- 商品多条码表（可选，按需创建）
-- =============================================
CREATE TABLE IF NOT EXISTS goods_barcode (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键 ID',
    tenant_id BIGINT NOT NULL COMMENT '租户 ID',
    sku_id BIGINT NOT NULL COMMENT '商品 SKU ID',
    barcode VARCHAR(50) NOT NULL COMMENT '商品条码',
    barcode_type TINYINT NOT NULL DEFAULT 1 COMMENT '条码类型：1-主条码，2-副条码',
    remark VARCHAR(200) COMMENT '备注',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    create_user BIGINT COMMENT '创建人 ID',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    update_user BIGINT COMMENT '修改人 ID',
    is_deleted TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除：0-未删除，1-已删除',
    UNIQUE KEY uk_tenant_barcode (tenant_id, barcode),
    INDEX idx_tenant_id (tenant_id),
```

## 4. 库存模型详细设计

### 4.1 库存字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| stock | INT | **可用库存**：实际可销售库存 = 物理库存 - 占用库存 |
| reserved_stock | INT | **预留/在途库存**：货未到但可预售的数量 |
| is_unlimited_stock | TINYINT | **是否无限库存**：1=无限，0=有限 |
| allow_negative_stock | TINYINT | **是否允许负库存**：1=允许，0=不允许 |
| warn_stock | INT | **库存预警值**：stock + reserved_stock <= warn_stock 时预警 |

### 4.2 库存计算公式

```
【可销售库存】
if is_unlimited_stock = 1:
    available_stock = ∞ (无限)
else:
    available_stock = stock + reserved_stock

【库存占用/扣减顺序】
1. 先扣 reserved_stock（预留库存）
2. reserved_stock 扣完后，再扣 stock（实际库存）
3. 如果 allow_negative_stock = 1，stock 可以扣到负数

【库存预警判断】
warning = (stock + reserved_stock) <= warn_stock
```

### 4.3 库存模式说明

| 模式 | is_unlimited | allow_negative | 适用场景 |
|------|--------------|----------------|---------|
| 标准库存模式 | 0 | 0 | 普通实物商品，库存不足时不能销售 |
| 负库存模式 | 0 | 1 | 允许超卖，后续补货 |
| 无限库存模式 | 1 | - | 服务类商品或虚拟商品 |
| 预售模式 | 0 | 0 | reserved_stock > 0，货未到先卖 |

## 5. 成本核算设计

### 5.1 成本价录入

- **商品编辑时必填**：cost_price 字段必填，默认 0
- **成本价变更记录**：变更时记录到 goods_change_log
- **前端显示**：商品列表、SKU 列表显示成本价（权限控制）

### 5.2 销售时成本快照

订单创建时，将商品成本价快照写入 `order_item.cost_price`：

```java
// 创建订单项时记录成本价
OrderItem item = new OrderItem();
item.setGoodsId(sku.getGoodsId());
item.setSkuId(sku.getId());
item.setGoodsName(goods.getGoodsName());
item.setPrice(sku.getPrice());        // 售价
item.setCostPrice(sku.getCostPrice()); // 成本价快照
item.setNum(num);
```

### 5.3 毛利计算

```
【单品毛利】
gross_profit = (price - cost_price) * num
gross_margin = (price - cost_price) / price * 100%

【订单毛利】
order_gross_profit = SUM(order_item.gross_profit)
```

## 6. 条形码设计

### 6.1 条码使用场景

1. **商品建档时录入条码**
   - 扫码枪扫描商品条码自动填充
   - 支持手工输入条码

2. **开单收银时扫码**
   - 支持扫码枪快速添加商品
   - 扫码后自动匹配 SKU（优先匹配 barcode，再匹配 sku_code）

3. **库存盘点时扫码**
   - 扫码盘点，提高效率

### 6.2 条码匹配逻辑

```java
/**
 * 根据条码查找 SKU
 * 匹配优先级：
 * 1. 精确匹配 barcode
 * 2. 精确匹配 sku_code
 * 3. 模糊匹配 barcode（可选）
 */
public GoodsSku getByBarcodeOrCode(String code, Long tenantId) {
    // 1. 先按 barcode 精确匹配
    GoodsSku sku = goodsSkuMapper.selectOne(
        new LambdaQueryWrapper<GoodsSku>()
            .eq(GoodsSku::getTenantId, tenantId)
            .eq(GoodsSku::getBarcode, code)
            .eq(GoodsSku::getStatus, 1)
            .last("LIMIT 1")
    );

    if (sku != null) {
        return sku;
    }

    // 2. 再按 sku_code 精确匹配
    return goodsSkuMapper.selectOne(
        new LambdaQueryWrapper<GoodsSku>()
            .eq(GoodsSku::getTenantId, tenantId)
            .eq(GoodsSku::getSkuCode, code)
            .eq(GoodsSku::getStatus, 1)
            .last("LIMIT 1")
    );
}
```

### 7.4 库存扣减核心逻辑

```java
/**
 * 扣减库存（新逻辑）
 */
@Override
@Transactional(rollbackFor = Exception.class)
public boolean deductStockWithReserve(Long skuId, int num) {
    GoodsSku sku = this.getById(skuId);
    if (sku == null) {
        throw new BusinessException(ErrorCode.SKU_NOT_FOUND);
    }

    // 1. 无限库存直接成功
    if (Integer.valueOf(1).equals(sku.getIsUnlimitedStock())) {
        return true;
    }

    int stock = sku.getStock() != null ? sku.getStock() : 0;
    int reservedStock = sku.getReservedStock() != null ? sku.getReservedStock() : 0;
    int totalStock = stock + reservedStock;
    boolean allowNegative = Integer.valueOf(1).equals(sku.getAllowNegativeStock());

    // 2. 检查库存是否足够（不允许负库存时）
    if (!allowNegative && totalStock < num) {
        throw new BusinessException(ErrorCode.STOCK_NOT_ENOUGH);
    }

    // 3. 计算扣减后的库存
    int remainNum = num;
    int newReserved = reservedStock;
    int newStock = stock;

    // 先扣预留库存
    if (newReserved > 0 && remainNum > 0) {
        if (newReserved >= remainNum) {
            newReserved -= remainNum;
            remainNum = 0;
        } else {
            remainNum -= newReserved;
            newReserved = 0;
        }
    }

    // 再扣实际库存
    if (remainNum > 0) {
        newStock -= remainNum;
    }

    // 4. 更新库存（使用 CAS 乐观锁）
    return goodsSkuMapper.updateStockWithReserve(
        skuId,
        newStock,
        newReserved,
        sku.getVersion()
    ) > 0;
}
```

**GoodsSkuMapper.java** 新增方法：
```java
@Update("UPDATE goods_sku " +
        "SET stock = #{newStock}, reserved_stock = #{newReserved}, version = version + 1 " +
        "WHERE id = #{skuId} AND version = #{version} AND is_deleted = 0")
int updateStockWithReserve(@Param("skuId") Long skuId,
                           @Param("newStock") int newStock,
                           @Param("newReserved") int newReserved,
                           @Param("version") Integer version);
```

```java
        Long userId = StpKit.SHOP.getUserId();

        return R.ok();
    }
}
```
