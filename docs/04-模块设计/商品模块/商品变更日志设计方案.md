# 商品变更日志设计方案

## 一、概述

本文档描述商品变更日志的设计与实现，包括数据存储格式、变更记录逻辑以及前端展示格式化。

## 二、数据库设计

### 2.1 表结构

```sql
CREATE TABLE `goods_change_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键 ID',
  `tenant_id` BIGINT NOT NULL COMMENT '租户 ID',
  `data_type` TINYINT NOT NULL COMMENT '数据类型：1-goods商品，2-goods_sku商品规格',
  `data_id` BIGINT NOT NULL COMMENT '数据 ID',
  `change_type` TINYINT NOT NULL COMMENT '变更类型：1-新增，2-修改，3-删除',
  `field_name` VARCHAR(64) DEFAULT NULL COMMENT '变更字段名（已废弃，保留兼容）',
  `before_value` TEXT COMMENT '变更前值（JSON格式，仅包含变更字段）',
  `after_value` TEXT COMMENT '变更后值（JSON格式，仅包含变更字段）',
  `batch_no` VARCHAR(64) DEFAULT NULL COMMENT '批次号',
  `remark` VARCHAR(255) DEFAULT NULL COMMENT '备注',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `create_user` BIGINT NOT NULL COMMENT '创建人 ID',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `update_user` BIGINT NOT NULL COMMENT '更新人 ID',
  `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '是否删除：0-否，1-是',
  PRIMARY KEY (`id`),
  KEY `idx_tenant_data` (`tenant_id`, `data_type`, `data_id`),
  KEY `idx_batch_no` (`tenant_id`, `batch_no`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品变更日志表';
```

## 三、存储格式

### 3.1 记录原则

- **一次操作一条记录**：保存商品时，无论修改了多少个字段（包括 SKU），都只记录一条日志
- **JSON 格式存储**：`before_value` 和 `after_value` 使用 JSON 存储完整变更数据
- **SKU 嵌套**：SKU 列表作为 `skuList` 字段嵌套在商品 JSON 中
- **只记录变更**：修改操作时，JSON 中仅包含发生变更的字段

### 3.2 忽略字段

以下字段不记录变更日志：

```java
- createTime
- createUser
- updateTime
- updateUser
- isDeleted
- serialVersionUID
```

## 四、前端展示格式化

### 4.1 设计目标

直接返回 JSON 给前端，用户体验不好，需要做以下优化：

1. **只展示变更字段**：不展示未变更的字段
2. **字段名翻译**：英文字段名 → 中文标签（`goodsName` → "商品名称"）
3. **枚举值翻译**：将代码值翻译为可读文本（`status=1` → "上架"）
4. **过滤内部字段**：隐藏技术字段（`version`、`tenantId`、`createTime` 等）
5. **SKU 友好展示**：标出哪个 SKU 的哪个字段发生了变更

### 4.2 字段映射配置

定义 `GoodsFieldEnum` 枚举，维护字段映射关系：

| 字段名 | 中文标签 | 值格式化 | 所属实体 |
|--------|----------|----------|----------|
| goodsName | 商品名称 | - | Goods |
| mainImage | 商品主图 | - | Goods |
| categoryId | 商品分类 | - | Goods |
| isService | 商品类型 | 0→实物商品, 1→服务商品 | Goods |
| status | 商品状态 | 0→下架, 1→上架 | Goods |
| specName | 规格名称 | - | GoodsSku |
| specValue | 规格值 | - | GoodsSku |
| price | 售价 | - | GoodsSku |
| costPrice | 成本价 | - | GoodsSku |
| stock | 库存 | - | GoodsSku |
| warnStock | 库存预警值 | - | GoodsSku |
| barcode | 商品条码 | - | GoodsSku |
| status | SKU状态 | 0→禁用, 1→启用 | GoodsSku |

### 4.3 内部字段过滤

以下内部字段不展示给用户：

```java
- id
- tenantId
- goodsId
- version
- createTime
- createUser
- updateTime
- updateUser
- isDeleted
```

### 4.4 响应数据结构

```java
// 单条变更日志
public class GoodsChangeLogVO {
    private Long id;
    private Integer dataType;
    private Long dataId;

    private String afterValue;   // 原始 JSON（保留兼容）
    private String batchNo;
    private String remark;
    private LocalDateTime createTime;
    private Long createUser;

    // 新增：格式化后的变更详情列表
    private List<GoodsChangeDetailVO> changeDetails;

// 单个字段变更详情
public class GoodsChangeDetailVO {
    private String fieldName;      // 字段名（英文）

    private String beforeValue;    // 变更前值（格式化后）
    private String afterValue;     // 变更后值（格式化后）

}
```

## 五、核心代码结构

### 5.1 文件清单

| 文件路径 | 说明 |
|----------|------|
| `entity/GoodsChangeLog.java` | 变更日志实体 |
| `dto/req/GoodsChangeLogReq.java` | 日志记录请求 DTO |
| `dto/resp/GoodsChangeLogVO.java` | 日志响应 VO |
| `dto/resp/GoodsChangeDetailVO.java` | 变更详情 VO |
| `service/GoodsChangeLogService.java` | 日志服务接口 |
| `service/impl/GoodsChangeLogServiceImpl.java` | 日志服务实现 |
| `service/impl/GoodsChangeLogFormatter.java` | 日志格式化组件 |
| `common/enums/GoodsFieldEnum.java` | 字段映射枚举 |
| `controller/pc/GoodsController.java` | 商品控制器 |
| `service/impl/GoodsServiceImpl.java` | 商品服务实现（记录日志） |

### 5.2 记录日志流程

```
saveGoods()
  ├─ createGoods() / updateGoodsIfChanged()
  ├─ saveSkuIncremental()
  │   ├─ createSku()
  │   ├─ updateSkuIfChanged()
  │   └─ deleteSku()
  └─ logCompleteChange()  ← 统一记录一条日志
       └─ goodsChangeLogService.logCreate() / logChange()
```

### 5.3 格式化日志流程

```
listChangeLog()
  └─ goodsChangeLogService.listByData()
       └─ goodsChangeLogFormatter.format()  ← 格式化每条日志
            ├─ parseJson()          ← 解析 JSON
            ├─ formatGoodsChanges() ← 处理商品字段
            └─ formatSkuChanges()   ← 处理 SKU 列表
```

## 六、后续维护指南

### 6.1 新增商品字段时

1. 在 `Goods.java` 中添加字段并写注释
2. 在 `GoodsFieldEnum` 中添加映射：
   ```java
   NEW_FIELD("newField", "新字段中文名称"),
   ```
3. 如果是枚举值，添加值格式化函数
4. 字段会自动参与变更对比和格式化展示

### 6.2 新增 SKU 字段时

1. 在 `GoodsSku.java` 中添加字段并写注释
2. 在 `GoodsFieldEnum` 中添加映射（注意区分与 Goods 字段）：
   ```java
   SKU_NEW_FIELD("newField", "SKU新字段中文名称"),
   ```
3. 字段会自动参与 SKU 变更对比和格式化展示

### 6.3 修改字段显示名称时

直接修改 `GoodsFieldEnum` 中对应枚举的 `label` 属性即可。

### 6.4 修改枚举值映射时

修改 `GoodsFieldEnum` 中对应枚举的 `valueFormatter` lambda 表达式。
