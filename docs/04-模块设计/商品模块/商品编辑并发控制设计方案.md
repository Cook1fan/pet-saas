# 商品编辑并发控制设计方案

> **文档导航：** [返回总览](./00-总览.md) | [核心业务逻辑](./03-核心业务逻辑.md)

---

## 1. 问题背景

### 1.1 场景描述

在商品编辑功能中，存在以下并发风险：

1. **多窗口编辑**：同一用户在多个浏览器窗口同时编辑同一个商品
2. **多用户编辑**：同一门店的不同管理员同时编辑同一个商品
3. **微服务架构**：未来服务拆分为微服务后，多实例部署时的并发问题

### 1.2 潜在问题

- 数据覆盖：后提交的请求覆盖先提交的请求
- SKU 重复创建：多次编辑可能导致 SKU 记录重复
- 数据不一致：商品基础信息与 SKU 信息不匹配

---

## 2. 设计目标

1. **防止并发冲突**：确保同一时间只有一个编辑操作能成功提交
2. **用户体验友好**：提供清晰的提示信息，避免用户困惑
3. **微服务就绪**：支持分布式环境下的并发控制
4. **性能影响最小**：在保证数据安全的前提下，尽量减少性能损耗

---

## 3. 技术方案

### 3.1 方案选型

采用 **分布式锁 + 乐观锁** 的双层防护方案：

| 方案 | 适用场景 | 说明 |
|-----|---------|------|
| Redisson 分布式锁 | 防并发提交 | 编辑操作开始前获取锁，确保串行执行 |
| 数据库乐观锁 | 防数据覆盖 | 通过版本号机制防止 ABA 问题 |

### 3.2 技术栈复用

项目已有的基础设施：
- **Redisson**：已配置，参考 `RedisConfig.java`
- **RedisKeyConstants**：已定义，用于统一管理锁键
- **BusinessException**：已定义，用于业务异常处理

## 4. 详细设计

### 4.1 Redis 键设计

在 `RedisKeyConstants` 中新增：

```java
/**
 * 商品编辑锁
 * 参数：tenantId, goodsId
 */
String GOODS_EDIT_LOCK_KEY = "goods:edit:lock:%d:%d";

/**
 * 商品版本号缓存
 * 参数：tenantId, goodsId
 */
String GOODS_VERSION_KEY = "goods:version:%d:%d";
```

### 4.2 数据库表变更

#### 4.2.1 goods 表新增字段

在 `goods` 表中新增乐观锁版本号字段：

```sql
ALTER TABLE goods ADD COLUMN version INT NOT NULL DEFAULT 0 COMMENT '乐观锁版本号' AFTER update_user;
```

对应的实体类 `Goods` 新增字段：

```java
/**
 * 乐观锁版本号
 */
@Version
private Integer version;
```

> **注意**：MyBatis-Plus 提供了 `@Version` 注解支持乐观锁，需在 `MybatisPlusConfig` 中配置乐观锁插件。

### 4.3 核心实现流程

#### 4.3.1 保存商品流程

```
开始
  ↓
获取分布式锁 (goods:edit:lock:{tenantId}:{goodsId})
  ↓
超时? → 抛出异常"商品正在被编辑，请稍后再试"
  ↓
查询当前商品信息
  ↓
版本号校验 (新增商品跳过)
  ↓
版本不匹配? → 抛出异常"商品已被修改，请刷新后重新编辑"
  ↓
保存商品基础信息 (version + 1)
  ↓
删除旧 SKU
  ↓
保存新 SKU
  ↓
更新 Redis 中的版本号
  ↓
释放分布式锁
  ↓
结束
```

#### 4.3.2 获取商品详情流程

在查询商品详情时，将版本号返回给前端：

```
查询商品信息
  ↓
查询 SKU 列表
  ↓
组装 VO，包含 version 字段
  ↓
返回给前端
```

前端在提交编辑时，需要将版本号原样传回。

### 5.3 MybatisPlusConfig 配置乐观锁

```java
@Configuration
public class MybatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

        // 多租户插件
        interceptor.addInnerInterceptor(new TenantLineInterceptor(new CustomTenantLineHandler()));

        // 分页插件
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));

        // 乐观锁插件
        interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());

        return interceptor;
    }
}
```

### 5.7 GoodsServiceImpl.saveGoods() 实现

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void saveGoods(GoodsSaveReq req, Long tenantId, Long userId) {
    // 编辑模式：获取分布式锁
    RLock lock = null;
    if (req.getId() != null) {
        String lockKey = String.format(RedisKeyConstants.GOODS_EDIT_LOCK_KEY, tenantId, req.getId());
        lock = redissonClient.getLock(lockKey);
        try {
            if (!lock.tryLock(3, 30, TimeUnit.SECONDS)) {
                throw new BusinessException(ErrorCode.GOODS_BEING_EDITED);
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new BusinessException(ErrorCode.INTERNAL_ERROR.getCode(), "系统异常");
        }
    }

    try {
        Goods goods;
        if (req.getId() == null) {
            // 新增商品
            goods = BeanConverter.convert(req, Goods.class);
            goods.setTenantId(tenantId);
            goods.setCreateUser(userId);
            goods.setUpdateUser(userId);
            goods.setVersion(0);
            if (goods.getStatus() == null) {
                goods.setStatus(1);
            }
            goodsMapper.insert(goods);
        } else {
            // 编辑商品
            goods = goodsMapper.selectById(req.getId());
            if (goods == null) {
                throw new BusinessException(ErrorCode.GOODS_NOT_FOUND);
            }

            // 乐观锁校验
            if (req.getVersion() == null || !req.getVersion().equals(goods.getVersion())) {
                throw new BusinessException(ErrorCode.GOODS_VERSION_MISMATCH);
            }

            Goods temp = BeanConverter.convert(req, Goods.class);
            temp.setId(goods.getId());
            temp.setTenantId(goods.getTenantId());
            temp.setCreateTime(goods.getCreateTime());
            temp.setCreateUser(goods.getCreateUser());
            temp.setUpdateUser(userId);
            temp.setVersion(goods.getVersion()); // MyBatis-Plus 会自动 +1

            // 使用 updateById 配合 @Version 实现乐观锁
            int updated = goodsMapper.updateById(temp);
            if (updated == 0) {
                throw new BusinessException(ErrorCode.GOODS_VERSION_MISMATCH);
            }

            goods = temp;
            goodsSkuService.deleteByGoodsId(goods.getId());
        }

        // 保存 SKU
        if (!CollectionUtils.isEmpty(req.getSkuList())) {
            for (GoodsSkuSaveReq skuReq : req.getSkuList()) {
                GoodsSku sku = BeanConverter.convert(skuReq, GoodsSku.class);
                sku.setId(null);
                sku.setTenantId(tenantId);
                sku.setGoodsId(goods.getId());
                sku.setCreateUser(userId);
                sku.setUpdateUser(userId);
                if (sku.getStatus() == null) {
                    sku.setStatus(1);
                }
                if (sku.getStock() == null) {
                    sku.setStock(0);
                }
                goodsSkuService.saveSku(sku);
            }
        }
    } finally {
        if (lock != null && lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}
```

## 6. 前端配合

### 6.1 接口响应

商品详情接口返回数据包含 `version` 字段：

```json
{
    "code": 0,
    "data": {
        "id": 1,
        "goodsName": "猫粮",
        "version": 5,
        "skuList": [...]
    }
}
```

### 6.2 编辑提交

前端在提交编辑时，需将 `version` 字段原样传回：

```json
{
    "id": 1,
    "goodsName": "猫粮（升级版）",
    "version": 5,
    "skuList": [...]
}
```

### 6.3 错误处理

| 错误码 | 提示文案 | 处理方式 |
|-------|---------|---------|
| 40010 | 商品正在被编辑，请稍后再试 | 显示提示，用户可选择重试 |
| 40011 | 商品已被修改，请刷新后重新编辑 | 显示提示，提供"刷新"按钮 |

## 7. 测试方案

### 7.1 并发测试场景

| 测试场景 | 预期结果 |
|---------|---------|
| 同一用户多窗口同时编辑同一商品 | 只有一个成功，其他提示"正在被编辑" |
| 不同用户同时编辑同一商品 | 只有一个成功，其他提示"正在被编辑" |
| A 用户编辑未提交，B 用户刷新后编辑 | B 用户获取到最新版本，可正常提交 |
| A 用户基于旧版本提交 | 提示"商品已被修改" |

### 7.2 性能测试

- 锁获取超时时间：3秒
- 锁持有超时时间：30秒
- 正常编辑操作耗时应小于 1 秒


## 9. 后续优化

### 9.1 编辑状态可视化

未来可考虑：
