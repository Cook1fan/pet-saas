# 商品图片压缩与流量优化方案

**文档导航：** [返回总览](./00-总览.md)

---

## 一、背景与问题

当前项目已实现阿里云OSS图片上传功能，但存在以下问题：

1. **图片大小差异大**：直接保存原图，小图几百KB，高清原图可达几MB
2. **OSS流量消耗快**：上传下载流量大，导致成本较高
3. **资源未充分利用**：OSS图片处理能力和CDN加速未启用
4. **用户体验待提升**：大图片加载慢，影响页面渲染速度

---

## 二、方案概述

采用 **"前端预压缩 + OSS实时处理 + CDN缓存 + 多规格图片"** 的四层优化策略：

| 层级 | 优化手段 | 目的 |
|------|---------|------|
| 前端层 | 上传前压缩（尺寸+质量） | 减少上传流量 |
| 存储层 | 原图保存 + OSS图片样式 | 实时生成多种规格 |
| 传输层 | CDN加速 + WebP格式 | 降低下载流量 |
| 展示层 | 场景化选择图片规格 | 提升加载速度 |

**预期效果**：OSS流量降低 70%+，图片加载速度提升 50%+

---

## 三、详细设计

### 3.1 前端预压缩（上传前）

#### 压缩策略

| 场景 | 目标尺寸 | 目标质量 | 格式 | 预期大小 |
|------|---------|---------|------|---------|
| 商品主图 | 最大边 1920px | 85% | WebP优先 | < 200KB |
| 商品详情图 | 最大边 1600px | 80% | WebP优先 | < 150KB |

#### 前端实现要点

- 使用 `canvas` + `Compressor.js`（或自建压缩逻辑）
- 优先转换为 WebP 格式（浏览器支持时）
- 保持宽高比，不拉伸图片
- 显示压缩前后对比（文件大小）
- 允许用户选择"保留原图"（但默认压缩）

#### 文件大小限制调整

- 压缩前最大 10MB（允许上传高清原图）
- 压缩后目标 < 500KB（建议值）

---

### 3.2 OSS 图片处理（存储层）

#### 图片样式定义

推荐使用 **URL 参数方式**（更灵活，无需在控制台配置）：

| 规格 | 用途 | OSS处理参数 |
|------|------|------------|
| 缩略图 | 商品列表页 | `?x-oss-process=image/resize,m_fill,w_200,h_200/quality,q_80/format,webp` |
| 小图 | 购物车/订单 | `?x-oss-process=image/resize,m_lfit,w_400,h_400/quality,q_85/format,webp` |
| 标准图 | 商品详情主图 | `?x-oss-process=image/resize,m_lfit,w_1200/quality,q_85/format,webp` |
| 原图 | 管理端查看 | 无参数 |

#### 后端代码修改

**修改 OssProperties.java**，添加图片处理配置：

```java
/**
 * CDN加速域名
 */
private String cdnDomain;

/**
 * 图片处理参数配置
 */
private ImageProcessProperties imageProcess = new ImageProcessProperties();

@Data
public static class ImageProcessProperties {
    /**
     * 缩略图参数：w_200,h_200,q_80
     */
    private String thumb = "image/resize,m_fill,w_200,h_200/quality,q_80/format,webp";

    /**
     * 小图参数：w_400,h_400,q_85
     */
    private String small = "image/resize,m_lfit,w_400,h_400/quality,q_85/format,webp";

    /**
     * 标准图参数：w_1200,q_85
     */
    private String normal = "image/resize,m_lfit,w_1200/quality,q_85/format,webp";
}
```

**修改 GoodsVO.java**，添加多规格图片URL字段：

```java
@Schema(description = "商品主图URL（原图）")
private String mainImage;

@Schema(description = "商品主图URL（缩略图）")
private String mainImageThumb;

@Schema(description = "商品主图URL（小图）")
private String mainImageSmall;

@Schema(description = "商品主图URL（标准图）")
private String mainImageNormal;
```

**修改 GoodsServiceImpl.java**，在转换VO时自动生成多规格URL：

```java
private GoodsVO convertToVO(Goods goods) {
    GoodsVO vo = new GoodsVO();
    BeanUtils.copyProperties(goods, vo);

    // 生成多规格图片URL
    if (StringUtils.hasText(goods.getMainImage())) {
        vo.setMainImage(goods.getMainImage());
        vo.setMainImageThumb(this.buildImageUrl(goods.getMainImage(), ossProperties.getImageProcess().getThumb()));
        vo.setMainImageSmall(this.buildImageUrl(goods.getMainImage(), ossProperties.getImageProcess().getSmall()));
        vo.setMainImageNormal(this.buildImageUrl(goods.getMainImage(), ossProperties.getImageProcess().getNormal()));
    }

    return vo;
}

private String buildImageUrl(String originUrl, String processParams) {
    if (!StringUtils.hasText(originUrl)) {
        return originUrl;
    }
    // 如果已经带参数，先去掉
    int queryIndex = originUrl.indexOf("?");
    String baseUrl = queryIndex > 0 ? originUrl.substring(0, queryIndex) : originUrl;
    return baseUrl + "?x-oss-process=" + processParams;
}
```

**修改 FileUploadServiceImpl.java**，生成 URL 时优先使用 CDN：

```java
// 构建完整访问URL（优先使用CDN）
String fileUrl;
if (StringUtils.hasText(ossProperties.getCdnDomain())) {
    fileUrl = String.format("%s/%s",
            ossProperties.getCdnDomain(),
            fileKey);
} else {
    fileUrl = String.format("https://%s.%s/%s",
            ossProperties.getBucket(),
            ossProperties.getEndpoint().replace("https://", ""),
            fileKey);
}
```

### 3.3 CDN 加速配置

#### OSS CDN 开启步骤

1. 在阿里云 OSS 控制台开启 CDN 加速
2. 配置 CDN 域名（如 `img.petsaas.com`）
3. 设置缓存规则：
   - 图片文件缓存 7 天
   - 带 `x-oss-process` 参数的 URL 视为不同资源独立缓存

#### 配置文件修改

**修改 application.yml**：

```yaml
aliyun:
  oss:
    # 原有配置...
    cdn-domain: ${ALIYUN_OSS_CDN_DOMAIN:}  # CDN域名，可选
```

---

### 3.4 防盗链配置

#### OSS Referer 白名单

在 OSS 控制台配置 Referer 白名单：

- 允许的域名：`*.petsaas.com`、`localhost`（开发环境）
- 允许空 Referer：是（方便直接访问图片调试）

#### （可选）URL 签名认证

对于敏感图片，使用 OSS 签名 URL，设置过期时间（当前代码已支持）。

## 四、实施步骤

### Phase 1：后端改造（优先实施）

1. **修改配置类** `OssProperties.java`
   - 添加 CDN 域名配置
   - 添加图片处理参数配置

2. **修改文件上传服务** `FileUploadServiceImpl.java`
   - 生成 URL 时优先使用 CDN 域名

3. **修改商品 VO** `GoodsVO.java`
   - 添加多规格图片字段

4. **修改商品服务** `GoodsServiceImpl.java`
   - 转换 VO 时自动生成多规格图片 URL

5. **修改配置文件** `application.yml`
   - 添加 CDN 域名配置项

### Phase 2：前端改造

1. **图片上传组件改造**
   - 集成图片压缩功能（Compressor.js 或 canvas）
   - 显示压缩前后大小对比
   - 支持 WebP 格式转换

2. **图片展示组件改造**
   - 列表页使用 `mainImageThumb`
   - 购物车/订单使用 `mainImageSmall`
   - 商品详情使用 `mainImageNormal`
   - 管理端可查看原图

### Phase 3：OSS/CDN 配置

1. 开启 OSS CDN 加速
2. 配置 CDN 域名和缓存规则
3. 配置 Referer 防盗链白名单
