# 宠物门店商品模块 MVP 设计方案（满足刚需+不过度设计+4周可上线）
核心原则：**先做「门店真实刚需的最小功能集」，不做复杂的多属性SKU，不做自定义分类，先跑通开单收银的核心闭环，后续根据种子客户反馈再迭代**。

---

## 一、先回答你的两个核心疑问
### 1. 要不要做SKU多规格？
**必须做简化版SKU，这是门店开单的刚需，但MVP阶段只做「单维度规格」，不做复杂的多属性组合**。

#### 为什么必须做？
宠物门店的真实场景里，90%的商品/服务都有多规格：
- 宠物食品：同一款猫粮，有1.5kg、5kg、10kg，价格差很多；
- 宠物用品：同一款猫砂盆，有S/M/L号，对应不同体型的猫；
- 宠物服务：同一款洗护，小型犬、中型犬、大型犬价格完全不一样；
- 宠物医疗：同一款疫苗，幼犬、成犬剂量不同，价格也不同。

如果没有多规格，门店开单时根本没法准确选价格、扣库存，系统就没法用。

#### 为什么只做「单维度规格」？
- **开发成本低**：单维度规格（比如「规格名称+规格值+价格+库存」）开发量极小，1-2天就能搞定；
- **门店够用**：95%的社区宠物门店，单维度规格完全能满足需求，不需要复杂的「颜色+尺寸+口味」多属性组合；
- **后续可扩展**：等有了种子客户反馈，真的有门店需要多属性组合，再迭代也不迟，MVP阶段先不做。

#### ③ 商品规格表（`goods_sku`）
> 存商品的规格、价格、库存，核心表，开单收银直接用这个表
| 字段名 | 类型 | 非空 | 说明 |
|--------|------|------|------|
| id | bigint | 是 | 主键，自增 |
| tenant_id | bigint | 是 | 租户ID |

| spec_name | varchar(50) | 是 | 规格名称（比如「规格」「重量」「体型」） |

| warn_stock | int | 否 | 库存预警值（实物商品用） |

| status | tinyint | 是 | 状态：0-禁用，1-启用 |
| create_time | datetime | 是 | 创建时间 |
| update_time | datetime | 是 | 更新时间 |

## 三、功能设计（MVP版，简单易上手）
### 1. 商品录入功能
#### 操作流程
1. 门店管理员/店员点击「商品管理」→「添加商品」；
2. 先选择**一级分类**→再选择**二级细分小类**（固定分类，直接下拉选择）；
3. 填写商品基础信息：商品名称、主图（可选）、是否服务类商品；
4. **添加规格**（核心步骤）：
   - 点击「添加规格」，输入规格名称（比如「重量」）；
   - 点击「添加规格值」，输入规格值（比如「1.5kg」）、售价、库存、库存预警值；
   - 可以继续添加多个规格值（比如「5kg」「10kg」）；
   - 如果是服务类商品，库存可以设为「不限」（stock=-1）；
5. 点击「保存」，商品和规格同时入库。

#### 界面设计要点
- 规格添加用「表格形式」，直观清晰，方便门店快速录入；
- 服务类商品自动隐藏「库存预警值」输入框，库存默认设为「不限」；
- 支持「复制规格」，比如同一款猫粮的不同规格，价格和库存可以快速修改。

---

### 2. 开单收银功能
#### 操作流程
1. 点击「开单收银」，进入开单页面；
2. 选择商品：
   - 可以通过「分类筛选」或「搜索商品名称」找到商品；
   - 点击商品后，**弹出规格选择弹窗**，显示该商品的所有规格（规格值、价格、库存）；
   - 选择规格后，自动带出价格和库存，输入购买数量；
3. 选择会员（可选）：输入手机号或扫描会员码，自动带出会员信息、储值余额、次卡；
4. 选择支付方式：微信支付、现金、储值、次卡；
5. 点击「确认开单」，自动扣减规格库存，生成订单和流水记录。

#### 界面设计要点
- 规格选择弹窗要大，字体清晰，方便店员快速选择；
- 库存不足时，规格显示「灰色」，无法选择；
- 服务类商品不显示库存，直接选择即可。

---

### 3. 库存管理功能
#### 核心功能
- **库存查看**：按商品/分类筛选，查看每个规格的库存、预警值；
- **库存预警**：首页显示「库存预警」模块，列出所有库存低于预警值的规格；
- **出入库管理**：选择规格，录入入库/出库数量、备注，自动更新规格库存。

---

## 四、MVP阶段的边界（避免过度设计）
1. **不做**多属性组合SKU（比如「颜色+尺寸+口味」），只做单维度规格；
2. **不做**门店自定义分类，先用固定分类，后续根据种子客户反馈再加；
3. **不做**商品条码扫描功能，后续迭代再加；
4. **不做**商品成本价、利润统计，后续迭代再加；
5. **不做**商品批量导入，先做手动录入，后续迭代再加Excel批量导入。

---

## 五、为什么这么设计？
1. **满足刚需**：简化版SKU+固定分类，完全能满足95%的社区宠物门店的开单收银需求；
2. **开发快**：数据库设计简单，功能逻辑清晰，1-2天就能搞定商品模块的开发；
3. **后续可扩展**：表结构预留了扩展空间，比如后续可以加`goods_attr`表做多属性组合，加`shop_goods_category`表做门店自定义分类；
4. **门店易上手**：固定分类+简单的规格添加，门店老板/店员不用培训，看一眼就会用。
