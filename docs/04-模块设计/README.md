## 为什么这些文档要放在根目录？

这些设计方案包含：
- **数据模型设计** - 前后端都需要理解数据结构
- **接口约定** - 前后端对接的基础
- **业务流程** - 前后端都需要理解完整业务逻辑
- **技术方案** - 如 Redis 预扣库存、分布式锁等，前端也需要理解配合

---

## 使用指南

### 开发新功能前

1. 在根目录打开 Claude 窗口
2. 阅读相关模块的设计方案
3. 在 `../01-API契约/` 下定义接口
4. 前后端确认后再开始开发

### 有设计变更时

1. 先更新本文档目录下的设计方案
2. 通知前后端开发人员
3. 再更新代码实现
