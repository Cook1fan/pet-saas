## 1. 业务概述

### 1.1 核心概念

| 概念 | 说明 |
|------|------|
| 储值规则 | 门店设置的储值优惠方案，如"储500送100" |
| 次卡规则 | 门店设置的次卡套餐，如"洗护10次999元，有效期1年" |
| 会员储值账户 | 会员在某门店的储值余额账户 |
| 会员次卡 | 会员购买的次卡，包含剩余次数和过期时间 |
| 储值购买记录 | 会员购买储值的历史记录 |
| 次卡购买记录 | 会员购买次卡的历史记录 |
| 储值消费记录 | 会员使用储值消费的记录 |
| 次卡核销记录 | 会员使用次卡核销的记录 |
| 核销码 | C端用户生成的用于次卡核销的6位数字码 |

### 1.2 业务规则

| 规则 | 说明 |
|------|------|
| 储值规则状态 | 0-禁用，1-启用，禁用后不可购买但不影响已购买的余额使用 |
| 次卡规则状态 | 0-禁用，1-启用，禁用后不可购买但不影响已购买的次卡使用 |
| 储值赠送金额 | 购买储值时立即到账，与本金同样可消费 |
| 次卡有效期 | 从购买日算起，加上规则设置的有效天数 |
| 核销码有效期 | 生成后15分钟内有效，超时失效 |
| 储值消费限制 | 余额不足时不能使用储值支付 |
| 次卡核销限制 | 次数为0或已过期时不能核销 |
| 退款规则 | 储值可退款，退本金不退赠送；次卡可退款，按比例折算 |

## 2. 核心流程设计

### 2.1 商家端：储值规则管理流程

```
商家操作流程：
1. 进入"储值规则管理"页面
2. 点击"创建规则"
3. 填写信息：
   - 规则名称（如"新手储值礼"）
   - 储值金额（如500）
   - 赠送金额（如100）
4. 保存规则，状态默认为"启用"
5. 可对规则进行编辑、启用/禁用、删除操作
6. C端小程序实时可见已启用的规则

时序图：
商家 → PC管理端 → 创建/编辑规则 → 后端校验 → 保存数据库 → 返回成功
```

### 2.2 商家端：次卡规则管理流程

```
商家操作流程：
1. 进入"次卡规则管理"页面
2. 点击"创建规则"
3. 填写信息：
   - 规则名称（如"洗护10次卡"）
   - 包含次数（如10）
   - 售价（如999）
   - 有效期天数（如365）
4. 保存规则，状态默认为"启用"
5. 可对规则进行编辑、启用/禁用、删除操作
6. C端小程序实时可见已启用的规则
```

### 2.3 C端：购买储值流程

```
用户操作流程：
1. 进入"我的储值"页面
2. 查看门店设置的储值规则列表
3. 选择一个储值规则
4. 确认购买信息，点击"立即购买"
5. 选择支付方式（微信支付）
6. 支付成功后，余额实时到账（本金+赠送）
7. 在"我的储值"中查看余额变动记录

核心逻辑：
┌─────────────────────────────────────────────────────────────┐
│ 1. 校验储值规则存在且已启用                                  │
│ 2. 查找或创建会员储值账户                                    │
│ 3. 创建储值购买记录（待支付）                                │
│ 4. 创建订单（order_info + order_item）                       │
│ 5. 调用微信支付，返回支付参数                                │
│ 6. 支付回调：更新订单状态，增加余额，记录流水                │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 C端：购买次卡流程

```
用户操作流程：
1. 进入"我的次卡"页面
2. 查看门店设置的次卡规则列表
3. 选择一个次卡规则
4. 确认购买信息，点击"立即购买"
5. 选择支付方式（微信支付）
6. 支付成功后，次卡实时到账
7. 在"我的次卡"中查看次卡详情和有效期

核心逻辑：
┌─────────────────────────────────────────────────────────────┐
│ 1. 校验次卡规则存在且已启用                                  │
│ 2. 创建会员次卡记录（待支付）                                │
│ 3. 创建订单（order_info + order_item）                       │
│ 4. 调用微信支付，返回支付参数                                │
│ 5. 支付回调：更新订单状态，激活次卡，设置过期时间            │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 商家端：储值消费/次卡核销流程

```
商家操作流程：
方式一：扫码核销（商家版小程序）
1. 进入商家版小程序"核销"页面
2. 扫描用户出示的核销码
3. 确认核销信息，点击"确认核销"
4. 核销成功，扣除次卡次数

方式二：PC端开单核销
1. 进入"开单收银"页面
2. 选择会员
3. 添加商品/服务
4. 选择支付方式：
   - 储值支付：校验余额，扣除金额
   - 次卡核销：选择次卡，校验次数，扣除次数
5. 完成开单，生成订单和核销记录
```

### 2.6 C端：生成核销码流程

```
用户操作流程：
1. 进入"我的次卡"页面
2. 选择一个有效的次卡（有剩余次数且未过期）
3. 点击"生成核销码"
4. 显示6位数字核销码和倒计时（15分钟）
5. 出示给商家扫码核销

核心逻辑：
┌─────────────────────────────────────────────────────────────┐
│ 1. 校验次卡存在且有效（有次数、未过期）                      │
│ 2. 检查是否存在未使用的核销码（避免重复生成）                │
│ 3. 生成6位数字核销码，设置15分钟过期时间                     │
│ 4. 保存核销码记录，关联会员次卡                              │
└─────────────────────────────────────────────────────────────┘
```

## 3. 数据库设计补充

### 3.1 新增表：recharge_order（储值购买记录表）

```sql
CREATE TABLE `recharge_order` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键 ID',
  `tenant_id` bigint NOT NULL COMMENT '租户 ID',
  `member_id` bigint NOT NULL COMMENT '会员 ID',
  `recharge_rule_id` bigint NOT NULL COMMENT '储值规则 ID',
  `order_no` varchar(32) NOT NULL COMMENT '订单号',
  `recharge_amount` decimal(10,2) NOT NULL COMMENT '储值金额',
  `give_amount` decimal(10,2) NOT NULL COMMENT '赠送金额',
  `pay_amount` decimal(10,2) NOT NULL COMMENT '实付金额',
  `pay_type` tinyint NOT NULL COMMENT '支付方式：1-微信，2-现金',
  `pay_status` tinyint NOT NULL DEFAULT 0 COMMENT '支付状态：0-待支付，1-已支付，2-已退款',
  `pay_time` datetime DEFAULT NULL COMMENT '支付时间',
  `transaction_id` varchar(64) DEFAULT NULL COMMENT '微信支付单号',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `create_user` bigint DEFAULT NULL COMMENT '创建人 ID',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
  `update_user` bigint DEFAULT NULL COMMENT '修改人 ID',
  `is_deleted` tinyint NOT NULL DEFAULT 0 COMMENT '逻辑删除：0-未删除，1-已删除',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_order_no` (`order_no`),
  KEY `idx_tenant_member` (`tenant_id`, `member_id`),
  KEY `idx_pay_status` (`pay_status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='储值购买记录表';
```

### 3.5 现有表确认

已有表已包含所需字段，无需修改：
- `recharge_rule` - 储值规则表
- `card_rule` - 次卡规则表
- `member_account` - 会员储值账户表
- `member_card` - 会员次卡表
- `verify_code_record` - 核销码记录表
- `order_info` - 订单表
- `order_item` - 订单明细表
- `flow_record` - 流水记录表

### 3.6 ER图

```
recharge_rule (储值规则表)
  ↓ 1:N
recharge_order (储值购买记录表)
  ↓ N:1
member_account (会员储值账户表)
  ↓ 1:N
recharge_consume_record (储值消费记录表)

card_rule (次卡规则表)
  ↓ 1:N

  ↓ N:1

  ↓ 1:N

  ↓ N:1

```

        // 1. 查询储值订单
        RechargeOrder rechargeOrder = rechargeOrderMapper.selectOne(
            new LambdaQueryWrapper&lt;RechargeOrder&gt;()
                .eq(RechargeOrder::getOrderId, orderId)
        );
        if (rechargeOrder == null || rechargeOrder.getPayStatus() != 0) {
            log.warn("储值订单不存在或已支付，orderId={}", orderId);
            return;

        // 2. 更新储值订单状态
        rechargeOrder.setPayStatus(1);
        rechargeOrder.setPayTime(LocalDateTime.now());
        rechargeOrderMapper.updateById(rechargeOrder);

        // 3. 查询或创建会员储值账户
        MemberAccount account = getOrCreateMemberAccount(rechargeOrder.getMemberId(), rechargeOrder.getTenantId());
        BigDecimal totalAmount = rechargeOrder.getRechargeAmount().add(rechargeOrder.getGiveAmount());
        BigDecimal beforeBalance = account.getBalance();
        BigDecimal afterBalance = beforeBalance.add(totalAmount);

        // 4. 更新账户余额
        account.setBalance(afterBalance);
        account.setTotalRecharge(account.getTotalRecharge().add(rechargeOrder.getRechargeAmount()));
        memberAccountMapper.updateById(account);

        // 5. 记录储值消费记录
        RechargeConsumeRecord record = new RechargeConsumeRecord();
        record.setTenantId(account.getTenantId());
        record.setMemberId(account.getMemberId());
        record.setMemberAccountId(account.getId());
        record.setType(1); // 1-储值
        record.setAmount(totalAmount);
        record.setBalanceBefore(beforeBalance);
        record.setBalanceAfter(afterBalance);
        record.setRemark("储值" + rechargeOrder.getRechargeAmount() + "，赠送" + rechargeOrder.getGiveAmount());
        rechargeConsumeRecordMapper.insert(record);

        // 6. 更新订单状态并创建流水
        OrderInfo order = orderInfoMapper.selectById(orderId);
        order.setPayStatus(1);
        order.setPayTime(LocalDateTime.now());
        orderInfoMapper.updateById(order);
        flowRecordService.createFlowRecord(order, 1); // 1-充值
    }

    @Transactional(rollbackFor = Exception.class)
    public void consumeBalance(Long memberId, BigDecimal amount, Long orderId, String remark, Long tenantId, Long operatorId) {
        // 1. 查询会员储值账户
        MemberAccount account = memberAccountMapper.selectOne(
            new LambdaQueryWrapper&lt;MemberAccount&gt;()
                .eq(MemberAccount::getTenantId, tenantId)
                .eq(MemberAccount::getMemberId, memberId)
        );
        if (account == null) {
            throw new BusinessException("会员储值账户不存在");
        }
        if (account.getBalance().compareTo(amount) &lt; 0) {
            throw new BusinessException("储值余额不足");
        }

        // 2. 扣除余额
        BigDecimal beforeBalance = account.getBalance();
        BigDecimal afterBalance = beforeBalance.subtract(amount);
        account.setBalance(afterBalance);
        memberAccountMapper.updateById(account);

        // 3. 记录消费记录
        RechargeConsumeRecord record = new RechargeConsumeRecord();
        record.setTenantId(tenantId);
        record.setMemberId(memberId);
        record.setMemberAccountId(account.getId());
        record.setOrderId(orderId);
        record.setType(2); // 2-消费
        record.setAmount(amount.negate());
        record.setBalanceBefore(beforeBalance);
        record.setBalanceAfter(afterBalance);
        record.setRemark(remark);
        record.setCreateUser(operatorId);
        rechargeConsumeRecordMapper.insert(record);
    }

            account.setBalance(BigDecimal.ZERO);
            account.setTotalRecharge(BigDecimal.ZERO);

            throw new BusinessException("次卡已过期");
        }

        // 2. 检查是否存在未使用的核销码
        VerifyCodeRecord existing = verifyCodeRecordMapper.selectOne(
            new LambdaQueryWrapper&lt;VerifyCodeRecord&gt;()
                .eq(VerifyCodeRecord::getTenantId, tenantId)
                .eq(VerifyCodeRecord::getMemberCardId, memberCardId)
                .eq(VerifyCodeRecord::getStatus, 0)
                .gt(VerifyCodeRecord::getExpireTime, LocalDateTime.now())
        );
        if (existing != null) {
            // 返回现有核销码
            long remainSeconds = ChronoUnit.SECONDS.between(LocalDateTime.now(), existing.getExpireTime());
            return VerifyCodeVO.builder()
                .verifyCode(existing.getVerifyCode())
                .expireTime(existing.getExpireTime())
                .remainSeconds(remainSeconds)
                .build();
        }

        // 3. 生成新核销码
        String verifyCode = VerifyCodeUtil.generateVerifyCode();
        LocalDateTime expireTime = LocalDateTime.now().plusMinutes(15);

        VerifyCodeRecord record = new VerifyCodeRecord();
        record.setTenantId(tenantId);
        record.setVerifyCode(verifyCode);
        record.setMemberCardId(memberCardId);
        record.setMemberId(memberId);
        record.setExpireTime(expireTime);
        record.setStatus(0);
        verifyCodeRecordMapper.insert(record);

        return VerifyCodeVO.builder()
            .verifyCode(verifyCode)
            .expireTime(expireTime)
            .remainSeconds(900L)
            .build();
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public CardVerifyResultVO verifyByCode(String verifyCode, Long tenantId, Long operatorId) {
        // 1. 查询核销码
        VerifyCodeRecord codeRecord = verifyCodeRecordMapper.selectOne(
            new LambdaQueryWrapper&lt;VerifyCodeRecord&gt;()
                .eq(VerifyCodeRecord::getTenantId, tenantId)
                .eq(VerifyCodeRecord::getVerifyCode, verifyCode)
        );
        if (codeRecord == null) {
            throw new BusinessException("核销码不存在");
        }
        if (codeRecord.getStatus() != 0) {
            throw new BusinessException("核销码已使用或已失效");
        }
        if (LocalDateTime.now().isAfter(codeRecord.getExpireTime())) {
            codeRecord.setStatus(2);
            verifyCodeRecordMapper.updateById(codeRecord);
            throw new BusinessException("核销码已过期");
        }

        // 2. 校验会员次卡
        MemberCard memberCard = memberCardMapper.selectById(codeRecord.getMemberCardId());
        if (memberCard == null) {
            throw new BusinessException("次卡不存在");
        }
        if (memberCard.getRemainTimes() &lt;= 0) {
            throw new BusinessException("次卡次数已用完");
        }
        if (LocalDateTime.now().isAfter(memberCard.getExpireTime())) {
            throw new BusinessException("次卡已过期");
        }

        // 3. 查询次卡规则
        CardRule cardRule = cardRuleMapper.selectById(memberCard.getCardRuleId());

        // 4. 扣除次数
        int beforeTimes = memberCard.getRemainTimes();
        int afterTimes = beforeTimes - 1;
        memberCard.setRemainTimes(afterTimes);
        memberCardMapper.updateById(memberCard);

        // 5. 更新核销码状态
        codeRecord.setStatus(1);
        codeRecord.setVerifyTime(LocalDateTime.now());
        codeRecord.setVerifyUserId(operatorId);
        verifyCodeRecordMapper.updateById(codeRecord);

        // 6. 记录核销记录
        CardVerifyRecord verifyRecord = new CardVerifyRecord();
        verifyRecord.setTenantId(tenantId);
        verifyRecord.setMemberId(memberCard.getMemberId());
        verifyRecord.setMemberCardId(memberCard.getId());
        verifyRecord.setCardRuleId(memberCard.getCardRuleId());
        verifyRecord.setVerifyCodeRecordId(codeRecord.getId());
        verifyRecord.setCardName(cardRule != null ? cardRule.getName() : "次卡");
        verifyRecord.setVerifyTimes(1);
        verifyRecord.setRemainTimesBefore(beforeTimes);
        verifyRecord.setRemainTimesAfter(afterTimes);
        verifyRecord.setCreateUser(operatorId);
        cardVerifyRecordMapper.insert(verifyRecord);

        // 7. 查询会员信息
        Member member = memberMapper.selectById(memberCard.getMemberId());

        // 8. 返回结果
        return CardVerifyResultVO.builder()
            .verifyRecordId(verifyRecord.getId())
            .memberName(member != null ? member.getName() : "")
            .cardName(cardRule != null ? cardRule.getName() : "次卡")
            .verifyTimes(1)
            .remainTimes(afterTimes)
            .verifyTime(LocalDateTime.now())
            .build();
    }
}

## 8. 功能使用说明

### 8.1 商家端使用指南

#### 8.1.1 如何设置储值规则

**步骤：**
1. 登录PC管理后台
2. 进入"会员管理" → "储值规则"
3. 点击"新建储值规则"
4. 填写规则信息：
   - 规则名称：给规则起一个易懂的名字，如"新手储值礼"
   - 储值金额：用户需要支付的金额，如500
   - 赠送金额：额外赠送的金额，如100
5. 点击"保存"
6. 规则默认为"启用"状态，C端用户立即可见

**使用技巧：**
- 可以设置多个规则，如"储300送50"、"储500送100"、"储1000送300"
- 通过赠送金额梯度，引导用户储值更多
- 规则禁用后，已购买的用户不受影响，只是新用户不能购买

#### 8.1.2 如何设置次卡规则

**步骤：**
1. 登录PC管理后台
2. 进入"会员管理" → "次卡规则"
3. 点击"新建次卡规则"
4. 填写规则信息：
   - 规则名称：给次卡起一个名字，如"精洗10次卡"
   - 包含次数：次卡包含多少次，如10
   - 售价：次卡售价，如999
   - 有效期天数：次卡的有效期，如365（1年）
5. 点击"保存"

**使用技巧：**
- 可以针对不同服务设置不同次卡，如"洗护卡"、"寄养卡"
- 设置合理的有效期，避免次卡长期占用
- 可以设置"不限次"的次卡（需修改代码支持）

#### 8.1.3 如何给会员储值（线下）

**场景：** 会员到店，用现金储值

**步骤：**
1. PC端进入"会员管理" → "会员列表"
2. 找到对应会员，点击"详情"
3. 点击"手动储值"
4. 选择储值规则或输入自定义金额
5. 选择支付方式（现金）
6. 确认储值

#### 8.1.4 如何使用储值开单

**步骤：**
1. 进入"开单收银"页面
2. 选择会员
3. 添加商品/服务
4. 选择支付方式 → "储值支付"
5. 确认金额，点击"完成开单"
6. 系统自动扣除储值余额，并生成消费记录

**注意事项：**
- 余额不足时会提示，需要先储值或选择其他支付方式
- 消费记录会实时同步到C端

#### 8.1.5 如何核销次卡（方式一：扫码核销）

**推荐使用商家版小程序扫码核销：**
1. 商家登录商家版小程序
2. 进入"核销"页面
3. 扫描用户出示的核销码
4. 确认核销信息，点击"确认核销"
5. 核销成功，次卡次数自动扣除

#### 8.1.6 如何核销次卡（方式二：PC端开单）

**步骤：**
1. 进入"开单收银"页面
2. 选择会员
3. 添加服务（如"宠物洗护"）
4. 选择支付方式 → "次卡核销"
5. 选择要使用的次卡
6. 确认核销，点击"完成开单"

#### 8.1.7 如何查看储值/次卡记录

**步骤：**
1. 进入"数据中心" → "储值记录"或"次卡记录"
2. 选择筛选条件：
   - 时间范围
   - 会员（可选）
   - 类型（储值/消费/退款）
3. 点击"查询"查看结果
4. 点击"导出"下载Excel表格

### 8.3 常见问题

#### 8.3.1 商家常见问题

**Q: 可以给储值设置有效期吗？**
A: 当前版本储值余额长期有效，不设过期时间。

**Q: 次卡过期了，没用完的次数怎么办？**
A: 可以在后台手动为会员延期或补次数（需开发管理员功能）。

**Q: 可以给不同会员设置不同的储值规则吗？**
A: 当前版本所有会员看到的规则是一样的，如需区分可以创建多个规则。

**Q: 用户用储值消费后，我需要和平台结算吗？**
A: 不需要，储值金额在您自己的微信商户号内，平台不碰资金。

#### 8.3.2 用户常见问题

**Q: 储值的钱可以退款吗？**
A: 可以联系商家退款，储值金额可以退，赠送金额不退。

**Q: 次卡买多了可以退吗？**
A: 可以联系商家协商退款，一般按剩余次数比例折算。

**Q: 我在A门店买的储值，在B门店能用吗？**
A: 不能，储值和次卡都是门店独立的，各门店之间不通用。

**Q: 核销码可以发给别人用吗？**
A: 可以，核销码不绑定个人，出示即可使用，建议妥善保管。

**Q: 我忘了小程序入口怎么办？**
A: 可以扫描门店的小程序码，或在微信下拉搜索门店名称。
