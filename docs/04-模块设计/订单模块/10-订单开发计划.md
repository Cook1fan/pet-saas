## 一、概述

本文档描述订单功能的开发范围、接口约定、前后端分工，用于指导前后端并行开发。

### 1.1 核心订单类型

| 订单类型 | 触发端 | 说明 |
|---------|-------|------|
| 普通订单（开单收银） | PC 端 | 门店店员扫码/选择商品开单 |
| 活动订单（拼团/秒杀） | C 端小程序 | C 端用户参与活动下单 |

### 1.2 核心支付方式

| 支付方式 | 说明 | 适用场景 |
|---------|------|---------|
| 微信支付 | 调起微信支付 | PC 端扫码、C 端小程序 |
| 现金 | 线下收款 | PC 端开单 |
| 储值余额 | 会员余额扣款 | PC 端、C 端 |
| 次卡 | 扣除次卡次数 | PC 端、C 端 |

## 二、订单数据模型

### 2.1 订单表 `order_info`

| 字段 | 类型 | 说明 | 前端注意 |
|-----|------|------|---------|
| `id` | Long | 订单 ID | 订单唯一标识 |
| `order_no` | String | 订单号 | 显示给用户，格式：`ORD{时间戳}{6位随机}` |
| `member_id` | Long | 会员 ID | 可选，散客单为空 |
| `total_amount` | BigDecimal | 总金额 | 商品原价总和 |
| `pay_amount` | BigDecimal | 实付金额 | 实际支付金额 |
| `pay_type` | Integer | 支付方式 | 1-微信，2-现金，3-储值，4-次卡 |
| `pay_status` | Integer | 支付状态 | 0-待支付，1-已支付，2-已退款 |
| `pay_time` | LocalDateTime | 支付时间 | |
| `transaction_id` | String | 微信支付单号 | 微信回调返回 |

### 2.2 订单明细表 `order_item`

| 字段 | 类型 | 说明 |
|-----|------|------|
| `order_id` | Long | 订单 ID |
| `goods_id` | Long | 商品 ID |
| `sku_id` | Long | SKU ID |
| `goods_name` | String | 商品名称（快照） |
| `num` | Integer | 数量 |
| `price` | BigDecimal | 单价（快照） |

### 2.3 活动订单表 `activity_order`

| 字段 | 类型 | 说明 |
|-----|------|------|
| `activity_id` | Long | 活动 ID |
| `order_id` | Long | 主订单 ID |
| `member_id` | Long | 会员 ID |
| `group_id` | Long | 拼团 ID（拼团用） |
| `status` | Integer | 活动订单状态：0-待支付，1-已支付，2-拼团成功，3-拼团失败 |

  "code": 200,
  "data": {
    "orderId": 100,
    "orderNo": "ORD20260402123456789012",
    "payAmount": 39.90,
    "wechatPayParams": {
      "timeStamp": "1617293619",
      "nonceStr": "5K8264ILTKCH16CQ2502SI8ZNMTM67VS",

**前端调起支付说明：**

拿到 `wechatPayParams` 后，调用微信小程序 `requestPayment` 接口：

```javascript
wx.requestPayment({
  timeStamp: wechatPayParams.timeStamp,
  nonceStr: wechatPayParams.nonceStr,
  package: wechatPayParams.package,
  signType: wechatPayParams.signType,
  paySign: wechatPayParams.paySign,
  success(res) {
    // 支付成功，跳转订单详情页
  },
  fail(res) {
    // 支付失败，提示用户
  }
});
```

## 四、支付回调处理

### 4.1 微信支付回调接口

**路径：** `/api/wechat-pay/callback`

**说明：** 微信支付成功后，微信服务器会 POST 回调此接口，后端更新订单状态为「已支付」。

**前端无需处理：** 回调是微信服务器 → 后端服务器的异步通知，前端只需监听 `requestPayment` 的 success/fail。

## 七、关键注意事项

### 7.1 前端注意事项

1. **订单号展示**：`order_no` 是唯一对用户展示的订单标识，不要展示 `id`
2. **金额显示**：所有金额字段（`totalAmount`、`payAmount`、`price`）都是 `BigDecimal`，前端保留 2 位小数
3. **时间格式**：后端返回 `LocalDateTime`，格式为 ISO 8601（如 `2026-04-02T10:30:00`），前端按需格式化
4. **支付状态轮询**：微信支付成功后，建议前端轮询订单详情接口确认状态（作为回调的兜底）
5. **防抖处理**：活动下单按钮点击后立即置灰，防止重复提交

### 7.2 后端注意事项

1. **幂等性**：支付回调、退款等操作必须保证幂等
2. **超卖防止**：活动下单使用 Redis 预扣库存 + Redisson 分布式锁
3. **数据一致性**：订单创建、库存扣减、流水记录在同一事务中
4. **多租户隔离**：所有订单查询自动过滤 `tenant_id`
5. **日志记录**：所有支付操作记录详细日志，方便排查问题
