## 一、模块概述

订单模块是宠物 SaaS 系统的核心模块，支持**门店 PC 端开单收银**和**C 端小程序活动下单**两种场景，支持微信支付、现金、储值余额、次卡四种支付方式。

### 1.1 核心功能

| 功能 | 触发端 | 说明 |
|-----|-------|------|
| 开单收银 | PC 端 | 门店店员选择商品/服务开单 |
| 活动下单 | C 端小程序 | 会员参与拼团/秒杀活动 |
| 订单查询 | 三端 | 订单列表、订单详情 |
| 订单退款 | PC 端 | 已支付订单退款 |
| 微信支付回调 | 后端 | 微信异步通知支付结果 |

### 1.2 核心数据表

| 表名 | 说明 | 关键字段 |
|-----|------|---------|
| `order_info` | 订单主表 | order_no, pay_status, pay_type |
| `order_item` | 订单明细表 | goods_name, price, num |
| `activity_order` | 活动订单关联表 | activity_id, group_id, status |
| `flow_record` | 流水记录表 | flow_no, type, transaction_id |

## 四、开单收银流程（PC 端）

### 4.1 整体流程

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  1. 选择商品    │ →  │  2. 选择会员    │ →  │  3. 选择支付方式│
└─────────────────┘    └─────────────────┘    └─────────────────┘
         ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 校验：库存 / 余额 / 次数                                 │
│  5. 创建订单：order_info + order_item                       │
│  6. 扣库存（实物商品）                                       │
│  7. 扣余额/次数（储值/次卡支付）                             │
│  8. 更新订单状态为「已支付」（现金/储值/次卡）               │
│  9. 生成流水记录 flow_record                                 │
└─────────────────────────────────────────────────────────────┘
         ↓
┌─────────────────┐
│  10. 返回结果   │
└─────────────────┘
```

### 2.2 详细步骤说明

#### 步骤 1：接收请求参数

**接口：** `POST /api/pc/order/create`

**请求参数：**
```java
{
  "memberId": 1001,                    // 可选，散客可为 null
  "items": [
    {
      "skuId": 2001,                   // 商品 SKU ID
      "num": 2                          // 数量
    }
  ],
  "payType": 1,                        // 1-微信，2-现金，3-储值，4-次卡
  "cardId": null                        // 次卡支付时必填
}
```

#### 步骤 2：商品校验与金额计算

**代码位置：** `OrderServiceImpl.createOrder():43-55`

**处理逻辑：**
```java
for (每个商品) {
    1. 查询 GoodsSku，校验是否存在
    2. 若 stock != -1（非服务类商品），校验库存是否充足
    3. 累加计算：totalAmount += price * num
}
```

#### 步骤 3：支付方式前置校验

**代码位置：** `OrderServiceImpl.createOrder():57-74`

| 支付方式 | payType | 校验内容 |
|---------|---------|---------|
| 微信支付 | 1 | 无前置校验 |
| 现金支付 | 2 | 无前置校验 |
| 储值支付 | 3 | 1. memberId 不能为空<br>2. 会员余额 ≥ 订单金额 |
| 次卡支付 | 4 | 1. cardId 不能为空<br>2. 次卡剩余次数 > 0<br>3. 次卡未过期 |

#### 步骤 4：创建主订单

**代码位置：** `OrderServiceImpl.createOrder():76-86`

**处理逻辑：**
1. 生成订单号：`OrderNoUtil.generateOrderNo()` → 格式 `ORD{yyyyMMddHHmmss}{6位随机数}`
2. 构造 `OrderInfo` 对象
3. 初始状态：`pay_status = 0`（待支付）
4. 插入 `order_info` 表

#### 步骤 5：创建订单明细 & 扣库存

**代码位置：** `OrderServiceImpl.createOrder():88-106`

**处理逻辑：**
```java
for (每个商品) {
    1. 构造 OrderItem，插入 order_item 表
       - 快照保存 goods_name、price（防止后续商品变更影响历史订单）
    2. 若 stock != -1（实物商品）：
       - 调用 goodsSkuService.deductStock(skuId, num) 扣减库存
}
```

#### 步骤 6：根据支付方式完成订单

**代码位置：** `OrderServiceImpl.createOrder():108-122`

| 支付方式 | 处理逻辑 |
|---------|---------|
| **现金 (2)** | 直接调用 `completeOrder()` |
| **储值 (3)** | 1. 调用 `memberAccountMapper.deductBalance()` 扣余额<br>2. 调用 `completeOrder()` |
| **次卡 (4)** | 1. 调用 `memberCardMapper.deductTimes()` 扣次数<br>2. 调用 `completeOrder()` |
| **微信 (1)** | 不立即完成，后续由微信回调触发 |

#### 步骤 7：完成订单（completeOrder）

**代码位置：** `OrderServiceImpl.completeOrder():129-136`

**处理逻辑：**
1. 更新订单状态：`pay_status = 1`（已支付）
2. 记录支付时间：`pay_time = now()`
3. 记录第三方交易单号（微信支付时）
4. 调用 `flowRecordService.createFlowRecord()` 生成流水记录

## 三、活动下单流程（C 端小程序）

### 3.1 整体流程

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  1. 选择活动    │ →  │  2. 确认数量    │ →  │  3. 点击下单    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 校验：活动状态 / 时间 / 限购                             │
│  5. Redis 预扣库存（防止超卖）                                │
│  6. 获取 Redisson 分布式锁（防止重复下单）                     │
│  7. 创建订单：order_info + order_item + activity_order       │
│  8. 调用微信支付统一下单 API                                  │
│  9. 返回微信支付参数给前端                                    │
└─────────────────────────────────────────────────────────────┘
         ↓
┌─────────────────┐    ┌─────────────────┐
│  10. 前端调起   │ →  │  11. 支付成功   │
│   微信支付      │    │   微信回调      │
└─────────────────┘    └─────────────────┘
```

### 3.2 详细步骤说明

#### 步骤 1-3：前端操作

**接口：** `POST /api/user/activity/order/create`

**请求参数：**
```java
{
  "activityId": 1,    // 活动 ID
  "num": 1            // 购买数量
}
```

#### 步骤 4：活动基础校验

**代码位置：** `UserServiceImpl.createActivityOrder():127-140`

**校验内容：**
1. 活动是否存在且属于当前租户
2. 活动状态是否为「进行中」（status = 1）
3. 当前时间是否在活动时间范围内
4. 购买数量是否超过限购数量

#### 步骤 5：Redis 预扣库存

**代码位置：** `UserServiceImpl.createActivityOrder():142-147`

**处理逻辑：**
```java
String stockKey = String.format("activity:stock:%d", activityId);
Long stock = stringRedisTemplate.opsForValue().decrement(stockKey, num);

if (stock == null || stock < 0) {
    // 库存不足，回滚
    stringRedisTemplate.opsForValue().increment(stockKey, num);
    throw new BusinessException("活动库存不足");
}
```

**为什么用 Redis 预扣库存？**
- 性能高：Redis 单线程、内存操作
- 防超卖：提前锁定库存，避免大量请求打到数据库

#### 步骤 6：获取 Redisson 分布式锁

**代码位置：** `UserServiceImpl.createActivityOrder():149-159`

**处理逻辑：**
```java
String lockKey = String.format("order:user:%d:activity:%d", memberId, activityId);
RLock lock = redissonClient.getLock(lockKey);

if (!lock.tryLock(3, 10, TimeUnit.SECONDS)) {
    // 获取锁失败，回滚库存
    stringRedisTemplate.opsForValue().increment(stockKey, num);
    throw new BusinessException("请勿重复下单");
}
```

**为什么用分布式锁？**
- 防止同一用户重复提交订单
- 锁 Key 格式：`order:user:{userId}:activity:{activityId}`
- 等待时间 3 秒，锁持有时间 10 秒

#### 步骤 7：创建订单

**代码位置：** `UserServiceImpl.createActivityOrder():161-191`

**处理逻辑：**
1. 创建 `order_info`：主订单
2. 创建 `order_item`：订单明细
3. 创建 `activity_order`：活动订单关联
   - `status = 0`（待支付）

#### 步骤 8-9：调用微信支付 & 返回

**代码位置：** `UserServiceImpl.createActivityOrder():193-200`

**处理逻辑：**
1. 调用 `wechatPayService.createMiniProgramPayOrder(order)` 获取支付参数
2. 构造返回结果：
   ```java
   {
     "orderId": 100,
     "orderNo": "ORD20260402123456789012",
     "payAmount": 39.90,
     "wechatPayParams": {
       "timeStamp": "1617293619",
       "nonceStr": "5K8264ILTKCH16CQ2502SI8ZNMTM67VS",
       "package": "prepay_id=wx1234567890abcdef",
       "signType": "RSA",
       "paySign": "abc123..."
     }
   }
   ```

#### 步骤 10-11：前端调起支付 & 微信回调

**前端调起支付：**
```javascript
wx.requestPayment({
  timeStamp: wechatPayParams.timeStamp,
  nonceStr: wechatPayParams.nonceStr,
  package: wechatPayParams.package,
  signType: wechatPayParams.signType,
  paySign: wechatPayParams.paySign,

  }

```

**微信回调处理：**
- 接口：`POST /api/wechat-pay/callback`
- 验证签名 → 查询订单 → 更新订单状态 → 生成流水 → 释放锁

```
│  5. 更新订单状态为「已退款」                                 │
│  6. 退回库存（实物商品）                                       │
│  7. 退回余额（储值支付）                                       │
│  8. 生成退款流水记录                                          │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 详细步骤

**接口：** `POST /api/pc/order/refund`

**请求参数：**
```java
{
  "orderId": 100,
  "refundAmount": 99.00,
  "reason": "商品质量问题"
}
```

**处理逻辑：** `OrderServiceImpl.refundOrder()`

1. 查询订单，校验：
   - 订单是否存在
   - 订单状态必须是「已支付」
2. 更新订单状态为「已退款」
3. 遍历订单项，退回库存（实物商品）
4. 若原支付方式是储值，退回余额
5. 生成退款流水记录

## 六、关键技术点

### 6.1 超卖防止

**方案：Redis 预扣库存 + 数据库乐观锁**

```java
// 1. Redis 预扣（快速）
Long stock = stringRedisTemplate.opsForValue().decrement(stockKey, num);
if (stock < 0) {
    stringRedisTemplate.opsForValue().increment(stockKey, num);
    throw new BusinessException("库存不足");
}

// 2. 数据库扣减（带乐观锁，兜底）
// SQL: UPDATE goods_sku SET stock = stock - #{num} WHERE id = #{id} AND stock >= #{num}
```

### 6.2 重复下单防止

**方案：Redisson 分布式锁**

```java
String lockKey = String.format("order:user:%d:activity:%d", memberId, activityId);
RLock lock = redissonClient.getLock(lockKey);

if (!lock.tryLock(3, 10, TimeUnit.SECONDS)) {
    throw new BusinessException("请勿重复下单");
}

try {
    // 创建订单逻辑
} finally {
    if (lock.isHeldByCurrentThread()) {
        lock.unlock();
    }
}
```

### 6.3 数据一致性
