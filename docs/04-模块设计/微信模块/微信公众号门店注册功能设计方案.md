# 微信公众号门店注册功能设计方案

## 一、背景与目标

### 1.1 背景
- 目前只有 platform 超级管理员可以创建 shop 门店账号
- 宠物店主需要联系平台管理员才能开通账号，流程繁琐
- 需要提供更优雅的自助注册方式

### 1.2 目标
- 宠物店主可通过微信公众号自助注册 shop 账号
- 保留 platform 账号创建 shop 账号的原有方式
- 支持两种注册方式并行

---

## 二、两种注册方式对比

| 维度 | Platform 创建 | 微信公众号自助注册 |
|-----|--------------|-------------------|
| 触发方 | Platform 管理员 | 宠物店主（微信用户） |
| 手机号 | 必填 | 可选（通过微信获取） |
| 密码发送 | 短信（预留） | 公众号消息 |
| 门店信息 | 创建时填写 | 授权后补充填写 |
| 账号状态 | 立即可用 | 补充信息后可用 |

---

## 三、业务流程设计

### 3.1 微信公众号自助注册流程

```
1. 宠物店主关注微信公众号
   ↓
2. 点击菜单「商家管理平台」
   ↓
3. 跳转至 H5 授权页面，请求微信授权
   ↓
4. 用户确认授权，获取微信 openid、unionid、昵称、头像
   ↓
5. 后端检查该微信是否已注册 shop 账号
   ├─ 已注册 → 直接登录，跳转至门店管理页面
   └─ 未注册 → 进入下一步
   ↓
6. 尝试获取微信手机号（需用户授权）
   ├─ 成功获取 → 使用该手机号作为 admin_phone
   └─ 获取失败 → admin_phone 留空，后续可补充
   ↓
7. 创建临时 shop 账号（status=2-待完善）
   ├─ 创建 sys_tenant 记录
   ├─ 创建 shop_admin 记录（username=临时标识，password=随机生成）
   └─ 创建 shop_config 记录
   ↓
8. 跳转至「完善门店信息」页面
   ↓
9. 用户填写门店名称、地址等信息
   ↓
10. 保存信息，更新 shop 账号状态为 status=1-启用
    ↓
11. 生成登录 token
    ↓
12. 进入门店管理平台
    ↓
13. 通过公众号发送账号密码给用户（用于 PC 端登录）
```

### 3.2 Platform 创建流程（保持不变）

```
1. Platform 管理员登录
   ↓
2. 填写门店名称、老板手机号、地址
   ↓
3. 校验手机号唯一性
   ↓
4. 创建 sys_tenant、shop_admin、shop_config
   ↓
5. 生成账号（手机号）和随机密码
   ↓
6. 发送短信通知（预留接口）
```

## 四、数据库设计变更

### 4.1 新增表：shop_wechat_bind（门店微信绑定表）

```sql
CREATE TABLE shop_wechat_bind (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键 ID',
    tenant_id BIGINT NOT NULL COMMENT '租户 ID',
    openid VARCHAR(100) NOT NULL COMMENT '微信 openid（公众号）',
    unionid VARCHAR(100) COMMENT '微信 unionid',
    nickname VARCHAR(100) COMMENT '微信昵称',
    avatar VARCHAR(500) COMMENT '微信头像',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    is_deleted TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除',
    UNIQUE KEY uk_openid (openid),
    INDEX idx_tenant_id (tenant_id)
) COMMENT='门店微信绑定表';
```

### 4.2 修改表：sys_tenant

在 `sys_tenant` 表中调整 `status` 字段的含义：

| status 值 | 说明 |
|----------|------|
| 0 | 禁用 |
| 1 | 启用（正常状态） |
| 2 | 待完善信息（微信注册未填门店信息） |

```sql
ALTER TABLE sys_tenant
MODIFY COLUMN status TINYINT NOT NULL DEFAULT 2 COMMENT '状态：0-禁用，1-启用，2-待完善';
```

## 五、接口设计

### 5.1 微信公众号相关接口（新增）

#### 5.1.1 微信公众号授权回调

| 项目 | 内容 |
|-----|------|
| 接口名称 | 微信公众号授权回调 |
| 请求方式 | GET |
| URL | /api/mp/wx/callback |
| 说明 | 微信 OAuth2 授权回调接口 |

**请求参数：**
| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| code | String | 是 | 微信授权 code |
| state | String | 是 | 状态参数 |

**响应数据：**
重定向至 H5 页面，携带临时 token

---

#### 5.1.2 微信注册预检查

| 项目 | 内容 |
|-----|------|
| 接口名称 | 微信注册预检查 |
| 请求方式 | GET |
| URL | /api/mp/shop/register/check |
| 说明 | 检查微信是否已注册门店 |

**请求参数：**
| 字段名 | 类型 | 必填 | 说明 |

**响应数据：**

|-------|------|------|
| registered | Boolean | 是否已注册 |
| tenantId | Long | 租户 ID（已注册时返回） |

#### 5.1.3 获取微信手机号

| 项目 | 内容 |

| 接口名称 | 获取微信手机号 |

**请求参数：**
| 字段名 | 类型 | 必填 | 说明 |

**响应数据：**
| 字段名 | 类型 | 说明 |

#### 5.1.4 提交门店注册信息

| 项目 | 内容 |

| 接口名称 | 提交门店注册信息 |

**请求参数：**
| 字段名 | 类型 | 必填 | 说明 |

## 六、核心实现逻辑

### 6.1 注册状态流转

```
微信授权
   ↓
创建临时账号 (status=2, username=mp_{openid})
   ↓
用户填写门店信息
   ↓
更新 status=1, username=手机号（如有）或 mp_{openid}
   ↓
发送公众号消息通知账号密码
```

### 6.2 账号生成规则

| 场景 | username | password |
|-----|----------|----------|
| Platform 创建 | 手机号 | 随机8位 |
| 微信注册（有手机号） | 手机号 | 随机8位 |
| 微信注册（无手机号） | mp_{openid} | 随机8位 |

### 6.3 密码通知方式

| 注册方式 | 通知方式 |
|---------|---------|
| Platform 创建 | 短信（预留接口） |
| 微信注册 | 公众号客服消息 |

## 十、安全考虑

1. **临时 token 有效期**：微信授权后的临时 token 有效期设置为 30 分钟
2. **密码仅显示一次**：注册成功后密码仅在响应中返回一次，后续不再展示
3. **openid 唯一性**：一个微信 openid 只能注册一个门店账号
4. **手机号唯一性**：一个手机号只能注册一个门店账号（与 Platform 创建方式共用校验）
5. **状态控制**：待完善状态的账号无法登录 PC 端，只能在 H5 中继续完善信息
