### 场景1：新用户注册（未注册过）

#### 步骤详解

**阶段一：微信授权（步骤1-9）**

```
   微信用户        微信服务器         后端服务          Redis          数据库
    |                 |                 |               |              |
    |--1. 点击授权链接----------------->|               |              |
    |                 |--2. 跳转授权页面->|               |              |
    |--3. 确认授权--->|                 |               |              |
    |                 |--4. 带code回调-->|               |              |
    |                 |                 |--5. 用code换微信用户信息     |
    |                 |                 |               |              |
    |                 |                 |--6. 生成mpToken            |
    |                 |                 |--7. 存Redis(30分钟过期)     |
    |                 |<--8. 重定向到H5--|               |              |
    |<--9. 显示测试页面-----------------|               |              |
```

**阶段一详细说明：**

| 步骤 | 说明 | 涉及代码/文件 |
|-----|------|--------------|
| **1** | **宠物店主在微信里点击授权链接**<br>这个链接是事先构造好的，格式是：<br>`https://open.weixin.qq.com/connect/oauth2/authorize?appid=xxx&redirect_uri=xxx&...` | 手动构造链接 |
| **2** | **微信服务器显示授权页面**<br>页面会显示："xxx公众号要获取你的头像、昵称等信息" | 微信服务器 |
| **3** | **用户点击「同意授权」**<br>用户确认同意授权给我们的应用 | 用户操作 |
| **4** | **微信服务器回调后端**<br>微信带着 `code` 参数跳转到我们的 `/api/mp/wx/callback` 接口<br>这个 `code` 是临时的，只能用一次，5分钟过期 | `MpShopAuthController.wxCallbackRedirect()` |
| **5** | **后端用 code 换微信用户信息**<br>调用微信 API，用 code 换取：<br>- openid（用户在这个公众号的唯一标识）<br>- unionid（用户在微信开放平台的唯一标识）<br>- nickname（昵称）<br>- avatar（头像）<br>- session_key（会话密钥，用于解密手机号）<br><br>**注意**：当前是 Mock 实现，直接生成了模拟数据 | `WxMpServiceImpl.getUserInfo()` |
| **6** | **生成 mpToken**<br>随机生成一个 32 位的字符串作为临时 token<br>这个 token 用来标识这次授权会话 | `MpShopRegisterServiceImpl.handleWxCallback()` |
| **7** | **存入 Redis**<br>把微信用户信息和 mpToken 关联起来存入 Redis<br>Key 格式：`mp:token:{mpToken}`<br>有效期：30 分钟<br>如果 30 分钟内没完成注册，需要重新授权 | Redis 操作 |
| **8** | **重定向到 H5 测试页面**<br>后端返回 302 重定向，浏览器跳转到测试页面<br>URL 带上 mpToken 参数：<br>`/shop-register-test.html?mpToken=xxx` | `MpShopAuthController.wxCallbackRedirect()` |
| **9** | **显示测试页面**<br>H5 页面加载，从 URL 参数中获取 mpToken 并保存 | `shop-register-test.html` |

**阶段二：检查注册状态（步骤10-14）**

```
    |                 |                 |               |              |
    |--10. 点击"检查注册"-------------->|               |              |
    |                 |                 |--11. 从Redis取临时数据       |
    |                 |                 |               |              |
    |                 |                 |--12. 查询shop_wechat_bind   |
    |                 |                 |<-----------------------------|
    |                 |                 |--13. 返回"未注册"           |
    |<--14. 显示"填写门店信息"---------|               |              |
```

**阶段二详细说明：**

| 步骤 | 说明 | 涉及代码/文件 |
|-----|------|--------------|
| **10** | **用户点击「检查注册状态」按钮**<br>H5 页面带着 mpToken 调用后端接口 | `shop-register-test.html.checkRegister()` |
| **11** | **从 Redis 取出临时数据**<br>根据 mpToken 从 Redis 中取出之前存的微信用户信息<br>（openid、unionid、nickname、avatar、sessionKey 等）<br>如果 Redis 里找不到，说明 token 过期了，需要重新授权 | `MpShopRegisterServiceImpl.checkRegister()` |
| **12** | **查询 shop_wechat_bind 表**<br>用 openid 查询数据库，看看这个微信是否已经注册过门店<br>这张表存的是「微信-门店」的绑定关系 | `ShopWechatBindMapper.selectOne()` |
| **13** | **返回「未注册」**<br>查询结果为空，说明这个微信还没注册过<br>返回给前端：`registered=false, needPhone=true/false` | `MpShopRegisterController.checkRegister()` |
| **14** | **显示「填写门店信息」页面**<br>H5 看到未注册，显示表单让用户填：<br>- 门店名称（必填）<br>- 门店地址（必填）<br>- 联系手机号（选填） | `shop-register-test.html` |

**阶段三：提交注册（步骤15-25）**

```
    |                 |                 |               |              |
    |--15. 填写门店信息提交------------>|               |              |
    |                 |                 |--16. 校验openid唯一性       |
    |                 |                 |--17. 校验手机号唯一性        |
    |                 |                 |               |              |
    |                 |                 |--18. 插入sys_tenant         |
    |                 |                 |----------------------------->|
    |                 |                 |--19. 插入shop_admin         |
    |                 |                 |----------------------------->|
    |                 |                 |--20. 插入shop_config        |
    |                 |                 |----------------------------->|
    |                 |                 |--21. 插入shop_wechat_bind   |
    |                 |                 |----------------------------->|
    |                 |                 |--22. 生成登录token          |
    |                 |                 |--23. 删除Redis临时数据      |
    |                 |                 |               |              |
    |                 |                 |--24. 发公众号消息(模拟)      |
    |<--25. 显示注册成功(含账号密码)---|               |              |
```

**阶段三详细说明：**

| 步骤 | 说明 | 涉及代码/文件 |
|-----|------|--------------|
| **15** | **用户填写表单并提交**<br>用户填写门店名称、地址、手机号（可选）<br>点击「提交注册」按钮 | `shop-register-test.html.submitRegister()` |
| **16** | **校验 openid 唯一性**<br>再次查询 `shop_wechat_bind` 表，确认这个微信还没注册<br>防止重复提交 | `ShopWechatBindMapper.selectOne()` |
| **17** | **校验手机号唯一性（如果有手机号）**<br>如果用户填了手机号，或者之前从微信获取了手机号<br>查询 `sys_tenant` 表，确认这个手机号没被其他门店用过<br>一个手机号只能注册一个门店 | `SysTenantMapper.selectCount()` |
| **18** | **插入 sys_tenant 表（创建门店）**<br>创建门店记录：<br>- shop_name: 门店名称<br>- admin_phone: 手机号（如果有）<br>- address: 门店地址<br>- status: 1（启用状态）<br>返回自增的 tenant_id | `SysTenantMapper.insert()` |
| **19** | **插入 shop_admin 表（创建管理员）**<br>创建门店管理员账号：<br>- tenant_id: 刚才生成的门店 ID<br>- username: 手机号（如果有），否则 `mp_{openid}`<br>- password: BCrypt 加密后的随机 8 位密码<br>- role: `shop_admin`（门店管理员）<br>- status: 1（启用） | `ShopAdminMapper.insert()` |
| **20** | **插入 shop_config 表（创建门店配置）**<br>创建门店配置记录：<br>- tenant_id: 门店 ID<br>- shop_name: 门店名称<br>- address: 门店地址<br>- phone: 联系电话 | `ShopConfigMapper.insert()` |
| **21** | **插入 shop_wechat_bind 表（绑定微信）**<br>创建微信与门店的绑定关系：<br>- tenant_id: 门店 ID<br>- openid: 微信 openid<br>- unionid: 微信 unionid<br>- nickname: 微信昵称<br>- avatar: 微信头像 | `ShopWechatBindMapper.insert()` |
| **22** | **生成登录 token**<br>用 Sa-Token 生成登录 token<br>登录身份是刚才创建的门店管理员<br>token 可以用于后续的 API 调用 | `StpKit.SHOP.login()` |
| **23** | **删除 Redis 临时数据**<br>注册成功了，清理掉临时的 mpToken 数据<br>释放 Redis 空间 | `redisTemplate.delete()` |
| **24** | **发送公众号消息（模拟）**<br>通过微信公众号给用户发消息，告知账号密码<br>当前是 Mock 实现，只打印日志<br>**注意**：实际需要接入微信公众号模板消息 API | `WxMpServiceImpl.sendRegisterSuccessNotification()` |
| **25** | **显示注册成功页面**<br>返回给前端：<br>- tenantId: 门店 ID<br>- token: 登录 token<br>- username: 管理员账号<br>- password: 初始密码（明文，仅这一次显示！）<br><br>**重要**：前端要提示用户「请妥善保存账号密码」 | `shop-register-test.html` |

## 接口调用顺序详解

### Controller.mp 下的接口关系

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    MpShopAuthController (认证)                          │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. GET  /api/mp/wx/callback       ← 微信授权回调入口                  │
│    └─→ handleWxCallback()          ← 生成mpToken并存Redis             │
│       └─→ 重定向到 /shop-register-test.html?mpToken=xxx              │
│                                                                          │
│ 2. POST /api/mp/shop/login          ← 老用户登录                      │
│    └─→ login()                                                          │
└─────────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────────┐
│                  MpShopRegisterController (注册)                        │
├─────────────────────────────────────────────────────────────────────────┤
│ 1. GET  /api/mp/shop/register/check  ← 检查注册状态                  │
│    └─→ checkRegister()                                                  │
│       ├─ 从Redis取mpToken对应的数据                                    │
│       └─ 查询shop_wechat_bind表                                        │
│                                                                          │
│ 2. POST /api/mp/shop/register/phone  ← 获取微信手机号(可选)          │
│    └─→ getPhone()                                                       │
│       └─ 解密微信手机号数据                                             │
│                                                                          │
│ 3. POST /api/mp/shop/register/submit  ← 提交注册信息                  │
│    └─→ submitRegister()                                                 │
│       ├─ 校验openid唯一性                                              │
│       ├─ 校验手机号唯一性(如果有)                                      │
│       ├─ 创建sys_tenant(门店)                                          │
│       ├─ 创建shop_admin(管理员)                                         │
│       ├─ 创建shop_config(门店配置)                                      │
│       ├─ 创建shop_wechat_bind(微信绑定)                                │
│       ├─ 生成登录token                                                  │
│       ├─ 删除Redis临时数据                                              │
│       └─ 发送公众号消息(模拟)                                           │
└─────────────────────────────────────────────────────────────────────────┘
```

## 关键文件职责

| 文件 | 职责 |
|------|------|
| **MpShopAuthController** | 处理微信授权回调、老用户登录 |
| **MpShopRegisterController** | 处理注册检查、提交注册等 |
| **MpShopRegisterService** | 核心业务逻辑实现 |
| **WxMpService** | 封装微信API调用（当前是Mock） |
| **MpAuthTempData** | Redis中存储的临时数据结构 |

## 数据库操作时序（新用户注册）

```
1. 查询 shop_wechat_bind (检查是否已注册)
   ↓
2. 插入 sys_tenant (创建门店)
   ↓
3. 插入 shop_admin (创建门店管理员)
   ↓
4. 插入 shop_config (创建门店配置)
   ↓
5. 插入 shop_wechat_bind (绑定微信)
```

---

## 核心数据流转

### mpToken 的一生

```
1. 微信授权回调时生成 (handleWxCallback)
   ↓
2. 存入Redis, key="mp:token:{mpToken}" (30分钟过期)
   ↓
3. 后续接口都用这个token从Redis取微信信息
   ↓
4. 注册成功后删除
```

### Redis 中存储的内容 (MpAuthTempData)

```java
{
  openid: "mp_openid_xxx",      // 微信openid
  unionid: "mp_unionid_xxx",    // 微信unionid
  nickname: "微信用户",           // 微信昵称
  avatar: "https://...",         // 微信头像
  sessionKey: "mock_...",        // 微信会话密钥(解密手机号用)
  phone: "13800138000",         // 手机号(如果获取了)
  tempTenantCreated: false,      // 是否已创建临时租户
  tempTenantId: null             // 临时租户ID
}
```

## 总结

**整个流程就是：**

1. **微信授权** → 拿到用户身份，生成临时token存Redis
2. **检查注册状态** → 用token查数据库看是否注册过
3. **分情况处理**：
   - 未注册 → 填门店信息 → 创建所有数据 → 返回账号密码
   - 已注册 → 直接登录 → 生成token
