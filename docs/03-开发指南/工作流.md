# 统一开发工作流

> 本文档定义前后端协同开发的标准流程

---

## 🎬 完整开发流程

### 阶段一：统一规划（根目录 Claude）

**在项目根目录打开 Claude 窗口，执行以下操作：**

1. **理解需求**
   - 阅读 `docs/00-总览/PRD.md`
   - 阅读 `docs/00-总览/技术方案.md`

2. **定义 API 契约**
   - 在 `docs/01-API契约/` 下创建接口定义
   - 编写 `types.ts` 定义共用类型
   - 前后端确认接口设计

3. **拆分任务**
   - 按模块拆分后端任务
   - 按页面拆分前端任务
   - 确定依赖关系和优先级

---

### 阶段二：并行开发

#### 后端开发（在 pet-saas-backend/ 打开 Claude）

```
1. 基于 API 契约实现 Controller
   ↓
2. 实现 Service 业务逻辑
   ↓
3. 实现 Mapper 数据访问
   ↓
4. 运行 Knife4j 验证接口
   ↓
5. 写单元测试
```

#### 前端开发（在 pet-saas-admin-pc/ 打开 Claude）

```
1. 基于 API 契约和 types.ts 生成 API 封装
   ↓
2. 用 Mock 数据开发页面组件
   ↓
3. 实现状态管理和路由
   ↓
4. 写组件测试
```

---

### 阶段三：联调对接

1. **后端完成接口** → 提供 Knife4j 文档地址
2. **前端替换 Mock** → 对接真实接口
3. **全链路测试** → 验证完整功能流程
4. **Bug 修复** → 闭环问题

---

## 🔄 旧流程 vs 新流程对比

| 环节 | 旧流程（有问题） | 新流程（推荐） |
|-----|----------------|---------------|
| **文档位置** | 分散在各项目 | 统一在根目录 `docs/` |
| **API 设计** | 后端先实现再给文档 | 前后端先一起定义契约 |
| **Claude 窗口** | 后端一个、前端一个 | 根目录一个 + 各子项目一个 |
| **开发顺序** | 后端做完 → 前端再做 | API 契约确定后并行开发 |
| **联调时机** | 全部做完才联调 | 按模块逐步联调 |

---

## 📌 Claude 窗口使用建议

### 窗口 1：根目录（统一规划用）
- **用途**：API 契约设计、跨项目协调、文档更新
- **何时打开**：新项目启动、新模块规划、跨端问题讨论

### 窗口 2：pet-saas-backend/（后端开发用）
- **用途**：Java 后端具体实现
- **何时打开**：写后端代码时

### 窗口 3：pet-saas-admin-pc/（前端开发用）
- **用途**：Vue3 前端具体实现
- **何时打开**：写前端代码时

### 窗口 4：pet-saas-miniapp/（小程序开发用）
- **用途**：小程序具体实现
- **何时打开**：写小程序代码时
