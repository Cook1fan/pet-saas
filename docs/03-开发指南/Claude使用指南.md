# Claude 使用指南

> 如何在本项目中高效使用 Claude 进行开发

---

## 🖥️ Claude 窗口配置

### 根目录窗口（统一规划）

**打开方式：**
- VS Code：打开整个 `pet-saas/` 文件夹
- IDEA：打开整个 `pet-saas/` 项目（或作为一个模块）

**适用场景：**
- ✅ 设计 API 契约
- ✅ 讨论跨端架构
- ✅ 更新统一文档
- ✅ 规划新功能模块

**常用指令：**
- `/plan` - 规划新功能的实现方案
- `/ecc:plan` - 调用 planner 代理做详细规划

---

### 后端窗口（pet-saas-backend/）

**打开方式：**
- IDEA：单独打开 `pet-saas-backend/` 项目

**适用场景：**
- ✅ 写 Java Controller/Service/Mapper
- ✅ 实现业务逻辑
- ✅ 调试后端问题
- ✅ 写后端测试

**常用指令：**
- `/ecc:tdd` - 测试驱动开发
- `/ecc:code-review` - 代码审查

---

### 前端窗口（pet-saas-admin-pc/）

**打开方式：**
- VS Code：单独打开 `pet-saas-admin-pc/` 文件夹

**适用场景：**
- ✅ 写 Vue3 组件
- ✅ 实现页面逻辑
- ✅ 调试前端问题
- ✅ 写前端测试

**常用指令：**
- `/ecc:feature-dev` - 功能开发
- `/ecc:code-review` - 代码审查

---

## 🔗 多 Claude 窗口协同技巧

### 场景：开发一个新功能

1. **根目录窗口**：
   ```
   "帮我设计会员模块的 API 契约，写到 docs/01-API契约/ 下"
   ```

2. **等 API 契约确定后**，同时打开：
   - **后端窗口**：基于契约实现接口
   - **前端窗口**：基于契约和 types.ts 用 mock 开发

3. **后端接口写完后**：
   - 前端窗口："把 mock 换成真实后端接口"

---

### 场景：遇到跨端问题

1. 在**根目录窗口**描述问题：
   ```
   "后端返回的订单数据格式，前端解析有问题，帮我看看：
   - 后端代码在 pet-saas-backend/...
   - 前端代码在 pet-saas-admin-pc/..."
   ```

2. 根目录 Claude 可以同时读取两个项目的代码，给出统一解决方案

---

## 📝 各项目的 CLAUDE.md

每个子项目都应该有自己的 `CLAUDE.md`，告诉 Claude：
- 技术栈细节
- 代码规范
- 项目结构
- 常用命令

示例内容参考各项目现有的 `CLAUDE.md`。

---

## 💾 数据库操作规范

### 🔴 重要规则：优先使用 MCP MySQL 工具

**遇到数据库相关问题时，按以下顺序操作：**

1. **先用 MCP MySQL 工具**（第一选择）：
   ```
   mcp__mysql-local__mysql_query - 参数 sql
   ```
   - 查询表结构：`SHOW CREATE TABLE table_name`
   - 创建表：直接写 `CREATE TABLE` 语句
   - 修改表：直接写 `ALTER TABLE` 语句
   - 插入数据：直接写 `INSERT` 语句

2. **不要做的事**：
   - ❌ 不要写 Java 代码来初始化数据库表
   - ❌ 不要通过后端启动逻辑来自动建表
   - ❌ 不要复杂的数据库迁移代码（除非是正式生产环境的版本管理）

3. **适用场景**：
   - ✅ 开发阶段表结构调整
   - ✅ 缺少表、字段时直接创建
   - ✅ 添加测试数据
   - ✅ 快速验证数据库问题

## ⚡ 效率提示

1. **善用 Plan 模式**：复杂功能先用 `/plan` 规划好再动手
2. **并行开发**：API 契约确定后，前后端可以同时开工
3. **及时文档**：有变更先更新 `docs/` 下的文档，再写代码
4. **代码审查**：写完代码立即用 `/ecc:code-review` 审查
5. **数据库优先 MCP**：有数据库问题先调用 MCP MySQL 工具
