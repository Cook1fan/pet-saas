# 回归测试清单

## 概述

回归测试是指在修改代码后，重新测试现有功能以确保没有引入新的缺陷。本清单包含项目核心功能的回归测试要点。

## 回归测试策略

### 1. 自动化回归测试
- 使用Playwright进行PC端E2E回归测试
- 使用UniApp Automator进行小程序端E2E回归测试
- 使用Postman/Newman进行API回归测试

### 2. 手动回归测试
- 核心业务流程重点测试
- 边缘场景和异常处理
- 跨模块功能集成测试

## 回归测试执行流程

### 1. 测试前准备
- 确保测试环境可用
- 准备测试数据
- 更新测试脚本

### 2. 自动化测试执行
```bash
# 运行PC端回归测试
cd docs/05-测试规范/e2e-script/pc
npm run test -- --grep="回归"

# 运行小程序端回归测试
cd ../miniapp
npm run test -- --grep="回归"

# 运行API回归测试
cd pet-saas-backend
mvn test -Dtest=RegressionTest
```

### 3. 手动测试重点
- 高优先级功能
- 复杂业务流程
- 边缘场景
- 异常处理

### 4. 测试报告
- 生成自动化测试报告
- 记录手动测试结果
- 分析测试结果，标记风险点
