# 测试快速开始

## 首次使用

### 1. 安装依赖

```bash
# 进入项目根目录
cd C:\Users\cook1\Desktop\pet-saas

# 安装PC端测试依赖
cd docs/05-测试规范/e2e-script/pc
npm install

# 安装小程序端测试依赖
cd ../miniapp
npm install

# 返回根目录
cd ../../..
```

### 2. 配置测试环境

```bash
# 复制PC端环境变量文件
cd docs/05-测试规范/e2e-script/pc
cp .env.example .env

# 复制小程序端环境变量文件
cd ../miniapp
cp .env.example .env
```

**修改 `.env` 文件内容**：
```env
# PC端环境变量
BASE_URL=http://localhost:5173
API_BASE_URL=http://localhost:8080
ADMIN_USERNAME=admin
ADMIN_PASSWORD=123456

# 小程序端环境变量
WECHAT_DEVTOOLS_PATH=C:/Program Files (x86)/Tencent/微信web开发者工具/cli.bat
MINIPROGRAM_PROJECT_PATH=C:/Users/cook1/Desktop/pet-saas/pet-saas-miniapp/dist/dev/mp-weixin
TEST_PHONE=13800138000
TEST_CODE=123456
```

### 3. 启动开发服务器

**后端服务**：
```bash
cd pet-saas-backend
mvn spring-boot:run
```

**前端服务**：
```bash
cd pet-saas-admin-pc
npm run dev
```

**小程序服务**：
```bash
cd pet-saas-miniapp
npm run dev:mp-weixin
```

## 快速执行测试

### 运行PC端测试

```bash
# 进入PC端测试目录
cd docs/05-测试规范/e2e-script/pc

# 安装Playwright浏览器
npx playwright install

# 运行所有测试
npm run test

# 运行特定测试文件
npm run test -- auth.spec.ts

# 运行带调试的测试
npm run test:debug

# 运行带UI界面的测试
npm run test:ui

# 生成并查看HTML报告
npm run test:ci
npm run report
```

### 运行小程序端测试

```bash
# 进入小程序测试目录
cd docs/05-测试规范/e2e-script/miniapp

# 运行所有测试
npm run test

# 运行CI模式（生成JSON报告）
npm run test:ci

# 生成HTML报告
npm run test:report
```

### 运行后端集成测试

```bash
cd pet-saas-backend

# 运行所有单元测试
mvn test

# 运行集成测试
mvn test -Dtest=**/integration/**/*Test.java

# 查看测试覆盖率报告
mvn jacoco:report
```

### 运行API测试

```bash
cd pet-saas-backend

# 使用Postman导入API测试
# API文档地址：http://localhost:8080/doc.html
```

## 常见问题

### 1. Playwright 安装失败

**问题**：安装 Playwright 浏览器时失败
**解决方案**：
```bash
# 清理缓存
npm cache clean --force

# 重新安装
npm install
npx playwright install --with-deps
```

### 2. 微信开发者工具自动化失败

**问题**：小程序测试无法启动微信开发者工具
**解决方案**：
1. 确保微信开发者工具已正确安装
2. 在微信开发者工具中开启自动化端口
3. 检查环境变量路径是否正确

### 3. 数据库连接失败

**问题**：集成测试时无法连接数据库
**解决方案**：
1. 检查数据库服务是否正常运行
2. 检查数据库连接配置
3. 确保测试数据库已创建

### 4. 测试超时

**问题**：某些测试超时
**解决方案**：
1. 检查网络连接
2. 检查服务器响应时间
3. 调整测试超时参数

## 测试优化建议

### 1. 提升测试速度

```bash
# 并行执行
npm run test -- --workers=4

# 只运行修改过的测试
npm run test -- --grep="最新"
```

### 2. 减少 Flaky 测试

- 使用稳定的选择器（如 `data-testid`）
- 避免硬编码的等待时间
- 使用智能等待机制
- 确保测试数据的独立性

### 3. 优化报告

```bash
# 生成Allure报告
npm install -D allure-playwright
npm run test -- --reporter=allure-playwright
npx allure generate allure-results -o allure-report
npx allure open allure-report
```
