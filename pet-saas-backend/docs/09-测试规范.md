## 9. 测试规范

### 9.1 测试策略

#### 9.1.1 测试金字塔

| 测试层级 | 占比 | 范围 | 执行速度 |
|---------|------|------|---------|
| 单元测试 | 70% | Service 层、工具类、算法 | 快（<5分钟） |
| 集成测试 | 20% | Controller 层、Mapper 层、外部服务 | 中（<15分钟） |
| 端到端测试 | 10% | 关键业务完整链路 | 慢 |

#### 9.1.2 测试框架选型

| 技术 | 版本 | 用途 |
|-----|------|------|
| JUnit 5 | 5.10.x | 测试框架基础 |
| Mockito | 5.x | Mock 框架，隔离外部依赖 |
| Spring Boot Test | 3.2.x | Spring 集成测试支持 |
| AssertJ | 3.24.x | 流式断言，提高可读性 |
| H2 Database | 2.x | 内存数据库，快速测试 |
| Testcontainers | 1.19.x | 容器化真实依赖（Redis/MySQL/RabbitMQ） |
| Awaitility | 4.2.x | 异步测试支持 |
| JaCoCo | 0.8.11 | 测试覆盖率统计 |

#### 9.1.3 必测 vs 选测

**必须 100% 覆盖**：
- 所有 Service 业务逻辑
- 多租户隔离逻辑
- 库存扣减、订单创建、支付回调等资金相关
- AI 额度控制
- 幂等性保证

**可选覆盖**：
- 简单 CRUD 操作
- 工具类中的纯函数

**不测试**：
- 第三方 API 实际调用（使用 Mock）
- 自动生成的代码

#### 9.6.2 订单创建测试

**测试场景矩阵**：

| 支付方式 | 会员 | 商品类型 | 库存状态 | 预期结果 |
|---------|------|---------|---------|---------|
| 微信支付 | 是 | 实物商品 | 充足 | 订单创建成功，库存扣减 |
| 微信支付 | 是 | 实物商品 | 不足 | 订单创建失败 |

#### 9.6.3 库存扣减并发测试

**测试目标**：验证高并发下不会超卖

```java
@Test
void shouldNotOversellUnderHighConcurrency() throws InterruptedException {
    // 给定：库存100
    int initialStock = 100;
    int threadCount = 200;
    CountDownLatch startLatch = new CountDownLatch(1);
    CountDownLatch endLatch = new CountDownLatch(threadCount);
    AtomicInteger successCount = new AtomicInteger(0);
    AtomicInteger failCount = new AtomicInteger(0);

    // 当：200个线程同时扣减库存
    for (int i = 0; i < threadCount; i++) {
        executor.submit(() -> {
            try {
                startLatch.await();
                boolean success = stockService.deductStock(goodsId, 1);
                if (success) successCount.incrementAndGet();
                else failCount.incrementAndGet();
            } catch (Exception e) {
                failCount.incrementAndGet();
            } finally {
                endLatch.countDown();
            }
        });
    }
    startLatch.countDown();
    endLatch.await();

    // 则：成功100次，失败100次，最终库存0
    assertEquals(100, successCount.get());
    assertEquals(100, failCount.get());
    assertEquals(0, stockService.getStock(goodsId));
}
```

#### 9.6.5 支付回调幂等性测试

**测试重点**：同一回调多次调用，结果一致

```java
@Test
void shouldBeIdempotentForSameCallback() {
    // 给定：支付回调数据
    // 当：第一次调用回调处理
    PaymentResult result1 = callbackService.handle(wxPayCallbackData);
    // 当：第二次调用相同回调
    PaymentResult result2 = callbackService.handle(wxPayCallbackData);

    // 则：两次结果一致，订单状态只更新一次，流水只生成一条
    assertEquals(result1, result2);
    verify(orderMapper, times(1)).updateStatus(any(), any());
    verify(flowRecordMapper, times(1)).insert(any());
}
```
