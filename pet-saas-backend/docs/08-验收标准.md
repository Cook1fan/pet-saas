## 9. 验收标准

### 9.1 平台 PC 管理端验收标准

| 功能点 | 验收标准 |
|-------|---------|
| 租户手动创建 | 1. 账号密码生成符合规则<br>2. 短信正常发送（需补充确认短信服务）<br>3. 租户登录后仅能查看本门店数据 |
| 租户状态管理 | 1. 禁用后租户所有端登录均被拦截<br>2. 重置密码后实时生效 |
| 租户列表查询 | 支持多条件组合筛选，列表展示租户核心信息 |
| 全局数据看板 | 数据实时更新，趋势图展示正常，无数据延迟 |

### 9.2 门店 PC 管理端验收标准

| 功能点 | 验收标准 |
|-------|---------|
| 门店信息完善 | 信息保存后实时生效，小程序端展示正确 |
| 员工账号管理 | 1. 店员账号仅能访问开放的功能<br>2. 账号禁用后无法登录 |
| 支付配置 | 配置完成后可正常发起支付、退款，资金直接进入门店商户号 |
| 会员手动录入 | 会员信息保存成功，支持按手机号/姓名模糊搜索 |
| 会员批量导入 | 1. 模板包含必填字段：手机号、姓名、宠物名、品种、服务周期<br>2. 导入时自动校验手机号格式、必填项，失败返回具体错误行号及原因<br>3. 校验通过后批量插入数据库，重复手机号自动跳过 |
| 宠物档案管理 | 宠物档案与会员绑定，提醒时间设置后可触发自动提醒 |
| 储值&次卡规则配置 | 规则保存后，C 端小程序实时展示，禁用后无法购买 |
| 核销管理 | 1. 余额/次数实时更新<br>2. 核销记录可查询，包含操作人、时间、金额 |
| 开单收银 | 1. 开单成功后生成唯一订单号<br>2. 商品库存自动扣减<br>3. 储值/次卡余额实时更新 |
| 订单管理 | 1. 筛选功能正常<br>2. 退款后余额/库存自动回退<br>3. 退款状态实时同步微信 |
| 商品录入 | 商品信息保存成功，可在开单时选择 |
| 库存预警 | 预警触发规则准确，提示清晰 |
| 拼团活动创建 | 1. 活动保存后 C 端实时展示<br>2. 活动时间到自动开始/结束<br>3. 小程序码带 tenant_id 和活动 ID，扫码直接进入活动 |
| 秒杀活动创建 | 同拼团活动验收标准，支持精确到分钟的活动时间设置 |
| 活动海报生成 | 1. 支持 3 种固定风格：温馨治愈/活泼促销/简约高级<br>2. 海报文字清晰、无错字，元素布局合理<br>3. 生成的海报可直接下载到本地 |
| AI 智能提醒 | 1. 提醒话术符合门店场景，语气亲切<br>2. 支持短信/微信模板消息两种发送方式（需补充确认）<br>3. 发送记录可查询 |
| AI 文案生成 | 1. 3 条文案风格有差异，符合宠物行业场景<br>2. 生成速度≤5 秒<br>3. 支持一键复制 |
| AI 问答配置 | 1. 常见问题回答 100% 匹配配置内容<br>2. 通用养护知识回答专业、易懂<br>3. 未知问题引导联系门店客服 |
| AI 调用额度管理 | 1. 额度统计准确，超出后拦截调用<br>2. 套餐购买后实时到账（套餐购买功能 P1） |
| 流水记录 | 流水记录 100% 准确，无遗漏 |
| 自动对账 | 1. 核对成功的流水标记为「已对账」<br>2. 金额/状态不一致的单边账标记为「异常」，触发告警 |
| 异常告警 | 告警触发及时，异常信息描述清晰 |

### 9.3 商家版小程序验收标准

| 功能点 | 验收标准 |
|-------|---------|
| 账号登录 | 登录成功，仅能查看本门店数据 |
| 数据概览 | 数据实时更新，与 PC 端一致 |
| 订单管理 | 列表展示完整，详情与 PC 端一致 |
| 次卡核销 | 核销成功后实时更新，有成功提示 |
| 海报生成 | 海报生成正常，可保存/转发 |

### 9.4 C 端微信小程序验收标准

| 功能点 | 验收标准 |
|-------|---------|
| 登录&门店绑定 | 1. 微信授权正常，自动同步会员信息<br>2. 扫码后自动绑定对应门店，无串店问题 |
| 多门店切换 | 切换成功后，数据实时更新为对应门店 |
| 活动中心 | 1. 活动展示与 PC 端配置一致<br>2. 支付流程正常，订单生成成功<br>3. 拼团成功/失败自动发送微信通知 |
| 个人中心 | 1. 数据与 PC 端一致，实时更新<br>2. 核销码生成正常，可被店员扫码核销 |
| AI 客服 | 1. 回答响应及时，内容准确<br>2. 历史对话可正常查看 |
| 服务提醒 | 消息接收正常，内容准确 |
