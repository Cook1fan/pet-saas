## 5.6 C 端微信小程序接口

### 5.6.1 微信授权登录

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 微信授权登录 | POST | /api/user/wxLogin | 微信授权登录 |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| code | String | 是 | 微信登录 code，通过 wx.login() 获取 |
| tenantId | Long | 否 | 租户 ID，扫码进入时传入，用于自动绑定门店 |

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| token | String | 登录凭证，C 端 token 前缀为 `user_` |
| member | Object | 会员信息对象 |
| &nbsp;&nbsp;├─ id | Long | 会员 ID |
| &nbsp;&nbsp;├─ phone | String | 手机号（已绑定手机号时返回） |
| &nbsp;&nbsp;├─ name | String | 姓名 |
| &nbsp;&nbsp;├─ openid | String | 微信 openid |
| &nbsp;&nbsp;└─ tenantId | Long | 当前租户 ID |
| isNewMember | Boolean | 是否新注册会员 |
| boundShopList | Array | 已绑定的门店列表（简略信息） |

**微信授权登录流程说明：**
1. 前端调用 wx.login() 获取临时 code
2. 前端调用后端 /api/user/wxLogin，传入 code 和 tenantId（可选）
3. 后端使用 code 调用微信 auth.code2Session 接口，换取 openid 和 session_key
4. 根据 openid 查询是否已存在会员：
   - 不存在：创建新会员，openid 存入 member 表
   - 存在：获取已有会员信息
5. 如果传入了 tenantId，检查是否已绑定该门店：
   - 未绑定：创建 member_shop_bind 绑定记录
   - 已绑定：更新最后访问时间
6. 使用 Sa-Token 登录，指定设备类型为 `user`
7. 将 tenantId、memberId、openid 存入 Session
8. 返回 token 和会员信息

### 5.6.2 已绑定门店列表

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 已绑定门店列表 | GET | /api/user/shop/list | 已绑定的门店列表 |

**请求参数：** 无

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| list | Array | 门店列表数组 |
| &nbsp;&nbsp;├─ tenantId | Long | 租户 ID |
| &nbsp;&nbsp;├─ shopName | String | 门店名称 |
| &nbsp;&nbsp;├─ address | String | 地址 |
| &nbsp;&nbsp;├─ phone | String | 联系电话 |
| &nbsp;&nbsp;├─ logo | String | logo URL |
| &nbsp;&nbsp;├─ isCurrent | Boolean | 是否当前选中门店 |
| &nbsp;&nbsp;└─ bindTime | LocalDateTime | 绑定时间 |

**多门店绑定逻辑说明：**
- 一个 C 端用户（openid）可以绑定多个门店
- 每次通过门店小程序码进入时，自动绑定该门店
- 用户可以在「我的门店」中切换当前门店
- 切换门店后，Session 中的 tenantId 更新为新门店的 ID
- 会员信息按租户隔离，每个门店的会员数据独立

### 5.6.6 活动下单

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 活动下单 | POST | /api/user/activity/order/create | 活动下单 |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| activityId | Long | 是 | 活动 ID |
| num | Integer | 是 | 数量 |

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| orderId | Long | 订单 ID |
| orderNo | String | 订单号 |
| payAmount | BigDecimal | 支付金额 |
| wechatPayParams | Object | 微信支付参数（用于前端调起支付） |

**活动下单流程说明：**
1. 验证活动是否存在且状态为进行中
2. 验证活动时间是否在有效范围内
3. 验证购买数量是否超过限购数量
4. Redis 预扣库存：`decrement activity:stock:{activityId}`
5. 若库存 < 0：回补库存，返回「活动库存不足」
6. 获取 Redisson 分布式锁：`order:user:{userId}:activity:{activityId}`
7. 若获取锁失败：返回「请勿重复下单」
8. 创建活动订单：插入 `activity_order` 表
9. 创建主订单：插入 `order_info`、`order_item` 表
10. 调用微信支付统一下单 API，获取支付参数
11. 返回订单信息和微信支付参数
12. 支付成功后，微信回调更新订单状态，释放分布式锁

### 5.6.11 生成核销码

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 生成核销码 | GET | /api/user/card/generateVerifyCode | 生成次卡核销码 |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| memberCardId | Long | 是 | 会员次卡 ID |

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| verifyCode | String | 核销码，6位数字 |
| expireTime | LocalDateTime | 过期时间，5分钟后过期 |
| cardName | String | 次卡名称 |
| remainTimes | Integer | 剩余次数 |

**核销码生成与校验机制说明：**
1. 验证会员次卡是否存在且属于当前会员
2. 验证次卡剩余次数是否大于0
3. 验证次卡是否在有效期内
4. 生成6位随机数字作为核销码
5. 创建核销记录：插入 verify_code_record 表
6. 核销记录包含：核销码、会员次卡ID、会员ID、租户ID、生成时间、过期时间、使用状态
7. 核销码有效期：5分钟
8. 同一会员次卡同一时间只能有一个有效的核销码（生成新码时旧码自动失效）

### 5.6.12 AI 客服对话

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| AI 客服对话 | POST | /api/user/ai/chat | AI 客服对话 |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| question | String | 是 | 用户问题 |

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| answer | String | AI 回答 |

**AI 客服对话流程说明：**
1. 检查门店 AI 调用额度：Redis 记录 `ai:limit:{tenantId}:{date}`
2. 若超出免费额度（50次/天）：返回「今日AI调用额度已用完」
3. 优先匹配门店配置的常见问题（预留功能）
4. 若匹配到：直接返回配置的标准答案
5. 若未匹配：构建 Prompt，包含门店信息、用户问题
6. 调用豆包大模型 API
7. 记录 AI 调用到 `ai_call_record` 表
8. 保存对话记录（可选，用于历史对话）
9. 返回 AI 回答
