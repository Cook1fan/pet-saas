## 5.5 商家版小程序接口

### 5.5.1 商家版登录

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 商家版登录 | POST | /api/merchant/login | 商家版小程序登录 |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| username | String | 是 | 账号，门店管理员或店员的登录账号 |
| password | String | 是 | 密码，BCrypt 加密存储 |

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| token | String | 登录凭证，商家版 token 前缀为 `merchant_` |
| admin | Object | 管理员信息对象 |
| &nbsp;&nbsp;├─ id | Long | 管理员 ID |
| &nbsp;&nbsp;├─ username | String | 账号 |
| &nbsp;&nbsp;├─ role | String | 角色：shop_admin-管理员，shop_staff-店员 |
| &nbsp;&nbsp;├─ status | Integer | 状态：0-禁用，1-启用 |
| &nbsp;&nbsp;└─ tenantId | Long | 租户 ID |

**业务逻辑说明：**
1. 根据账号查询管理员信息
2. 验证密码是否正确（BCrypt 校验）
3. 验证账号状态是否启用
4. 验证租户是否存在且状态正常
5. 使用 Sa-Token 登录，指定设备类型为 `merchant`
6. 将 tenantId、adminId、role 存入 Session
7. 返回 token 和管理员信息

### 5.5.2 数据概览

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 数据概览 | GET | /api/merchant/dashboard/overview | 数据概览 |

**请求参数：** 无

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| todayOrderCount | Integer | 今日订单数，统计今日已支付的订单数量 |
| todayGmv | BigDecimal | 今日 GMV，统计今日已支付订单的实付金额总和 |
| monthlyGmv | BigDecimal | 本月 GMV，统计本月已支付订单的实付金额总和 |

**数据统计逻辑：**
- 今日订单数：`pay_status = 1 AND DATE(pay_time) = 今天`
- 今日 GMV：`SUM(pay_amount) WHERE pay_status = 1 AND DATE(pay_time) = 今天`
- 本月 GMV：`SUM(pay_amount) WHERE pay_status = 1 AND YEAR(pay_time) = 今年 AND MONTH(pay_time) = 本月`
- 所有统计均按当前登录用户的 tenantId 进行租户隔离

### 5.5.4 次卡核销

| 接口名称 | 请求方式 | URL | 说明 |
|---------|---------|-----|------|
| 次卡核销 | POST | /api/merchant/card/verify | 次卡核销 |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| verifyCode | String | 是 | 核销码，6位数字，5分钟有效期 |

**响应数据：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| cardName | String | 次卡名称 |
| memberName | String | 会员姓名 |
| remainTimes | Integer | 核销后剩余次数 |
| expireTime | LocalDateTime | 次卡过期时间 |

**次卡核销业务逻辑说明：**
1. 根据核销码查询核销记录，验证核销码是否存在
2. 验证核销码是否已使用，已使用则返回错误
3. 验证核销码是否过期（有效期5分钟），过期则返回错误
4. 查询关联的会员次卡信息
5. 验证次卡剩余次数是否大于0
6. 验证次卡是否在有效期内
7. 扣减次卡剩余次数1次
8. 标记核销码为已使用，记录核销时间和核销人
9. 创建核销流水记录
10. 返回核销结果
