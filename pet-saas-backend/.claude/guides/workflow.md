## Phase 1: 工程初始化与基础设施（P0）

### 1.1 创建 Spring Boot 工程
- 使用 Spring Initializr 创建项目
- 配置 Maven pom.xml 依赖（包含测试依赖）

### 1.2 核心配置
- `application.yml` 多环境配置（dev/test/prod）
- MyBatis-Plus 多租户插件配置
- Sa-Token 多端登录配置
- Knife4j 接口文档配置
- Redis 配置

### 1.3 通用基础类
- `R<T>` 统一响应封装
- `GlobalExceptionHandler` 全局异常处理
- 通用常量、枚举定义

### 1.4 测试基础设施（与业务代码同步）
- 创建测试目录结构 `src/test/java/` 和 `src/test/resources/`
- 编写测试基类：`BaseUnitTest.java`、`BaseIntegrationTest.java`、`BaseControllerTest.java`
- 编写测试工具类：`TestDataBuilder.java`
- 配置 `application-test.yml` 测试环境
- 配置 pom.xml 测试依赖

### 1.5 数据库初始化
- 创建数据库 `pet_saas`
- 执行建表 SQL（按 `docs/05-数据库设计/`）
- 插入初始化数据（平台管理员、默认配置）

**验证标准**:
- [ ] 项目启动成功，无报错
- [ ] 访问 Knife4j 文档页面正常
- [ ] 数据库连接正常
- [ ] 单元测试目录结构完整
- [ ] 测试基类创建完成
