# 全端通用代码评审规范

---

## 通用评审检查清单

### 代码质量
- [ ] 代码可读且命名良好
- [ ] 函数聚焦（<50 行）
- [ ] 文件内聚（<800 行）
- [ ] 无深层嵌套（>4 层）
- [ ] 错误处理显式
- [ ] 无硬编码密钥或凭据
- [ ] 无 console.log 或调试语句
- [ ] 新功能有测试
- [ ] 测试覆盖率满足 80% 最低要求

### 安全审查
- [ ] 认证或授权代码已审查
- [ ] 用户输入已验证
- [ ] 数据库查询使用参数化（防止 SQL 注入）
- [ ] XSS 防护已启用（HTML 已净化）
- [ ] CSRF 保护已启用
- [ ] 认证/授权已验证
- [ ] 所有端点启用速率限制
- [ ] 错误消息不泄露敏感数据

---

## 后端专属评审（Java / SpringBoot）

### 分层架构
- [ ] 返回值规范：是否直接返回 Entity 实体对象，未转换为 VO 对象返回
- [ ] 配置注入规范：是否使用 @Value 注解，而非推荐的 @ConfigurationProperties 注解
- [ ] 日志打印规范：是否使用 System.out.println 打印日志，而非 @Slf4j 日志框架
- [ ] 方法参数规范：方法入参数量是否超过 2 个
- [ ] 私有方法调用规范：调用私有方法时，是否添加 this. 前缀
- [ ] Service 层规范：Service 层业务方法是否添加注释说明
- [ ] 字符串判断规范：字符串判断是否使用 StrUtil 工具类

### 安全与权限
- [ ] 所有对外接口必须加 @PreAuthorize 权限注解
- [ ] 所有入参必须做校验（@Valid、@Validated）
- [ ] 禁止在 Controller 层写业务逻辑

---

## 前端专属评审（Vue3 / TypeScript）

### 类型安全
- [ ] 禁止使用 `any` 类型，必须定义具体类型
- [ ] 所有 API 响应必须定义类型接口
- [ ] 所有 Props 必须定义类型接口

### API 请求
- [ ] 所有 API 请求必须封装在 `/src/api/` 目录下
- [ ] 页面组件禁止直接调用 axios，必须通过 API 封装层

### 组件规范
- [ ] 组件文件使用 PascalCase 命名
- [ ] 组合式函数使用 `use` 前缀
- [ ] 禁止在组件内写超过 200 行的逻辑，抽取为 composables

---

## 小程序专属评审（UniApp）

### 包体积优化
- [ ] 图片资源必须压缩后上传 CDN，禁止直接放入项目
- [ ] 按需引入组件，避免全量引入 UI 库
- [ ] 合理使用分包，非首屏页面放入分包
- [ ] 主包体积 < 2MB，单个分包 < 2MB

### 性能要求
- [ ] 列表必须使用虚拟滚动
- [ ] 图片使用 `lazy-load` 懒加载
- [ ] 避免频繁 setData，批量更新数据
